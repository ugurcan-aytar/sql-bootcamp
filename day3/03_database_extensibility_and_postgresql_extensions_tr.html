Gun 3: Veritabani Genisletilebilirligi ve PostgreSQL Uzantilari

Oturum 3: Veritabani Genisletilebilirligi ve PostgreSQL Uzantilari (2 saat)

Veritabani Genisletilebilirligi Kavrami

Modern veritabani sistemlerinin, belirli uygulama gereksinimlerini karsilamak icin genellikle temel SQL standartlarinin ve yerlesik ozelliklerin otesine gecmesi gerekir. Veritabani genisletilebilirligi, bir RDBMS tarafindan temel veritabani motoru kodunu degistirmeden yeni islevsellik, veri turleri, operatorler, indeksleme yontemleri veya prosedural diller eklemek icin saglanan mekanizmalari ifade eder.

Neden Bir Veritabanini Genisletelim?

  • Alana Ozgu Ihtiyaclar: CBS, biyoinformatik, finans vb. gibi belirli alanlar icin ozellestirilmis veri turleri veya fonksiyonlar eklemek.
  • Performans Optimizasyonu: Belirli is yukleri icin optimize edilmis ozel indeks turleri veya fonksiyonlar uygulamak.
  • Entegrasyon: Harici sistemler veya kutuphanelerle baglanti kurmak.
  • Yeni Ozellikler: Henuz standart SQL'in veya temel RDBMS'in parcasi olmayan islevselligi dahil etmek (ornegin, gelismis kriptografik fonksiyonlar, bulanik metin arama).
  • Kod Yeniden Kullanilabilirligi: Karmasik mantigi yeniden kullanilabilir fonksiyonlar veya prosedurler halinde paketlemek.

Veritabanlarinda Yaygin Genisletilebilirlik Mekanizmalari:

Uygulama detaylari buyuk olcude degismekle birlikte, yaygin yaklasimlar sunlardir:

  1. Kullanici Tanimli Fonksiyonlar (UDF) ve Sakli Prosedurler: Cogu veritabani, kullanicilarin SQL, prosedural diller (PL/pgSQL, T-SQL, PL/SQL gibi) veya hatta harici diller (Python, Java, C#, C) kullanarak ozel fonksiyonlar ve prosedurler yazmasina izin verir.
  2. Kullanici Tanimli Turler (UDT): Mevcut turlere dayali veya bilesik yapilar olarak ozel veri turleri olusturmak.
  3. Kullanici Tanimli Agregatlar (UDA): Ozel agregat fonksiyonlari olusturmak.
  4. Ozel Operatorler: Belirli veri turleri veya mantik icin yeni operatorler tanimlamak.
  5. Eklenti/Uzanti Mimarileri: Ilgili islevselligi paketlemek ve yuklemek icin resmi sistemler (ornegin, PostgreSQL Uzantilari, MySQL Eklentileri).
  6. Yabanci Veri Sarmalayicilari (FDW): Veritabani disinda bulunan verilere yerel tablolarmis gibi erismek icin mekanizmalar (SQL/MED standardinin parcasi, PostgreSQL'de iyi desteklenmektedir).
  7. Harici Dil Entegrasyonu: Veritabani icinden C, Java, Python vb. dillerde yazilmis kodu cagirmak (ornegin, SQL Server CLR Entegrasyonu, Oracle Harici Prosedurler, PostgreSQL PL/Python, PL/Perl, PL/Java).

Farkli RDBMS'lerde Genisletilebilirlik (Kisa Genel Bakis)

  • PostgreSQL: Saglam bir Uzanti sistemine sahiptir. Uzantilar, turler, fonksiyonlar, operatorler, indeks turleri vb. ekleyebilen paketlenmis modullerdir. Ornekler arasinda PostGIS (mekansal), pg_trgm (bulanik arama), pgcrypto (kriptografi) bulunur. Ayrica cesitli dillerde UDF'leri destekler (PL/pgSQL, PL/Python, PL/Perl, PL/Tcl, PL/Java, C, vb.) ve FDW'ler.
  • Oracle: Prosedural mantik icin buyuk olcude PL/SQL'e, Paketler'e (ilgili prosedurler, fonksiyonlar, turleri gruplama), Java Sakli Prosedurler'e ve Harici Prosedurler'e (C kodu cagirma) dayanir.
  • SQL Server: Prosedural mantik icin T-SQL, CLR Entegrasyonu (yonetilen .NET kodunun veritabani icinde calismasina izin verir) ve Harici Betikler (ornegin, Python, R) kullanir.
  • MySQL: SQL'de Sakli Prosedurler/Fonksiyonlar, C/C++ ile yazilmis Kullanici Tanimli Fonksiyonlar (UDF) ve daha kapsamli eklemeler icin bir Eklenti API'si destekler (depolama motorlari veya tam metin ayristiricilari gibi).
  • Db2: SQL PL, Java Sakli Prosedurler ve harici UDF'leri destekler.

PostgreSQL Uzantilari: Bir Ornek Calisma

Bu calistay PostgreSQL kullandigi icin, guclu uzanti sistemine odaklanacagiz. Uzantilar, onemli yetenekler eklemek icin temiz, surumlu ve yonetilebilir bir yol saglar.

PostgreSQL'de Uzantilari Yonetme:

  • Kullanilabilirlik: Uzantilar once sunucu dosya sistemine yuklenmelidir (genellikle isletim sistemi paket yoneticisi araciligiyla, ornegin apt install postgresql-14-postgis-3).
  • Listeleme: SELECT name, default_version, comment FROM pg_available_extensions; bir veritabanina yuklemeye uygun uzantilari gosterir.
  • Yukleme: CREATE EXTENSION extension_name [SCHEMA schema_name] [VERSION version_number]; uzantinin nesnelerini (tablolar, fonksiyonlar, turler) mevcut veritabanina, genellikle belirtilen veya varsayilan bir sema icine yukler.
  • Guncelleme: ALTER EXTENSION extension_name UPDATE [TO version_number]; yuklenmis bir uzantiyi daha yeni bir surume yukseltir (yeni surumun dosyalarinin sunucuya yuklenmis olmasi gerekir).
  • Kaldirma: DROP EXTENSION extension_name [CASCADE]; uzantinin nesnelerini veritabanindan kaldirir.

Yaygin ve Faydali PostgreSQL Uzantilari:

(Not: Bunlar PostgreSQL'in genisletilebilirlik modelinin gucunu gosteren PostgreSQL'e ozgu orneklerdir)

  1. pg_stat_statements: Tum SQL sorgulari icin yurutme istatistiklerini izler. Performans izleme ve ayarlama icin gereklidir.

    -- Yavas veya sik sorgular icin pg_stat_statements gorunumunu sorgula
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;
SELECT query, calls, total_exec_time, rows FROM pg_stat_statements ORDER BY total_exec_time DESC LIMIT 10;

  2. pgcrypto: Kriptografik fonksiyonlar saglar (hashleme, sifreleme/sifre cozme).

    -- Kriptografik fonksiyonlari kullanma
CREATE EXTENSION IF NOT EXISTS pgcrypto;
SELECT crypt("password123", gen_salt("bf"));
SELECT encrypt("sensitive data", "secret_key", "aes");

  3. uuid-ossp: Cesitli turde Evrensel Benzersiz Tanimlayicilar (UUID) olusturur.

    CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
-- Rastgele bir UUID olustur (Surum 4)
SELECT uuid_generate_v4();
-- Birincil anahtar icin varsayilan deger olarak kullan
CREATE TABLE items (
    item_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name TEXT
);

Yaygin ve Faydali PostgreSQL Uzantilari:

(Not: Bunlar PostgreSQL'in genisletilebilirlik modelinin gucunu gosteren PostgreSQL'e ozgu orneklerdir)

  1. hstore: Anahtar-deger deposu veri turu saglar. Yari yapilandirilmis veriler icin kullanislidir (ancak JSONB artik genellikle tercih edilmektedir).

    CREATE EXTENSION IF NOT EXISTS hstore;CREATE TABLE products_legacy (
    id INT,
    attributes HSTORE
);
INSERT INTO products_legacy VALUES (1, 'color=>red, size=>large');
SELECT attributes -> 'color' FROM products_legacy WHERE id = 1;

  2. pg_trgm: Bulanik metin arama ve benzerlik olcumu icin trigram eslemesini destekler.

    CREATE EXTENSION IF NOT EXISTS pg_trgm;
-- Benzer isimleri bul
SELECT name, similarity(name, 'PostgresQL')
FROM products
WHERE name % 'PostgresQL';
-- Performans icin *_trgm_ops kullanan bir GIN veya GiST indeksi gerektirir
CREATE INDEX idx_products_name_trgm
ON products USING gin (name gin_trgm_ops);

Yaygin ve Faydali PostgreSQL Uzantilari:

(Not: Bunlar PostgreSQL'in genisletilebilirlik modelinin gucunu gosteren PostgreSQL'e ozgu orneklerdir)

  1. ltree: Hiyerarsik agac yapilarini temsil etmek ve sorgulamak icin veri turu (ornegin, kategoriler, organizasyon semalari).

    CREATE EXTENSION IF NOT EXISTS ltree;
CREATE TABLE categories (
    id INT,
    path LTREE
);
INSERT INTO categories VALUES (1, 'Top.Electronics.Computers.Laptops');
-- Electronics'in tum alt ogelerini bul
SELECT * FROM categories WHERE path <@ 'Top.Electronics';

  2. tablefunc: Tablolar donduren fonksiyonlar saglar, ozellikle pivot tablolar olusturmak icin crosstab. 

    CREATE EXTENSION IF NOT EXISTS tablefunc;
-- Ornek: Satis verilerini yila gore pivotla
SELECT * FROM crosstab(
    'SELECT product, year, sales FROM sales_summary ORDER BY 1,2',
    'SELECT DISTINCT year FROM sales_summary ORDER BY 1'
) AS ct (product TEXT, year_2021 NUMERIC, year_2022 NUMERIC);

Yaygin ve Faydali PostgreSQL Uzantilari:

(Not: Bunlar PostgreSQL'in genisletilebilirlik modelinin gucunu gosteren PostgreSQL'e ozgu orneklerdir)

  1. intarray: Tamsayi dizilerini manipule etmek icin operatorler ve fonksiyonlar saglar (etiket sistemleri, izinler icin kullanisli).

    CREATE EXTENSION IF NOT EXISTS intarray;
-- Hem 1 VE 3 ile etiketlenmis ogeleri bul
SELECT item_id FROM items_with_tags WHERE tags @> ARRAY[1, 3];
-- 2 VEYA 4 ile etiketlenmis ogeleri bul
SELECT item_id FROM items_with_tags WHERE tags && ARRAY[2, 4];

  2. PostGIS: En onemli mekansal veritabani uzantisi, cografi nesneler, sorgular ve analiz icin kapsamli destek ekler.

    CREATE EXTENSION IF NOT EXISTS postgis;
CREATE TABLE locations (
    id INT,
    name TEXT,
    geom GEOMETRY(Point, 4326)
);
INSERT INTO locations VALUES (
    1,
    'City Hall',
    ST_SetSRID(ST_MakePoint(-74.0, 40.7), 4326)
);
-- Belirli bir mesafe icindeki konumlari bul
SELECT name
FROM locations
WHERE ST_DWithin(
    geom,
    ST_SetSRID(ST_MakePoint(-74.1, 40.7), 4326)::geography,
    5000
);

Uzantilari Secme ve Kullanma: Uzantilar guclu ozellikler eklerken, su hususlari goz onunde bulundurun:

  • Ihtiyac: Yalnizca gercekten ihtiyac duydugunuz uzantilari yukleyin.
  • Guvenilirlik: PostgreSQL ile birlikte gelen veya guvenilir kaynaklardan gelen uzantilari tercih edin.
  • Bakim: Uzantinin aktif olarak bakildigindan emin olun.
  • Performans Etkisi: Bazi uzantilar performansi etkileyebilir; uygun sekilde izleyin.
  • Bagimliliklar: Uzantilar arasindaki bagimliliklarin farkinda olun.

Veritabani genisletilebilirligi, veritabanini belirli ihtiyaclara gore uyarlamaniza olanak tanir, ancak dikkatli kullanilmalidir. PostgreSQL'in uzanti sistemi, bunu basarmak icin ozellikle iyi entegre edilmis ve guclu bir mekanizma saglar.
