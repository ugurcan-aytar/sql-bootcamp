Toplama Fonksiyonlari (Standart SQL ve Uzantilar)

Toplama fonksiyonlari bir satir kumesi uzerinde hesaplamalar yapar ve tek bir ozet deger dondurur. Raporlama ve veri analizi icin cok onemlidirler.

Yaygin Standart SQL Toplama Fonksiyonlari:

  1. COUNT(): Satirlari veya NULL olmayan degerleri sayar.

    • COUNT(*): Bir gruptaki toplam satir sayisini sayar.
    • COUNT(sutun_adi): sutun_adi NULL olmayan satirlarin sayisini sayar.
    • COUNT(DISTINCT sutun_adi): sutun_adi icindeki benzersiz NULL olmayan degerlerin sayisini sayar. 
    -- Standart SQL Ornekleri
SELECT COUNT(*) FROM orders; -- Toplam siparis
SELECT COUNT(shipped_date) FROM orders; -- Gonderilmis siparisler
SELECT COUNT(DISTINCT customer_id) FROM orders; -- Siparis veren benzersiz musteri sayisi

  2. SUM(sayisal_sutun): NULL olmayan sayisal degerlerin toplamini hesaplar.

    -- Standart SQL Ornekleri
SELECT SUM(order_total) FROM orders WHERE status = "completed"; -- Tamamlanmis siparislerin toplami

  1. AVG(sayisal_sutun): NULL olmayan sayisal degerlerin ortalamasini hesaplar. 

    -- Standart SQL Ornekleri
SELECT AVG(order_total) FROM orders WHERE status = "completed"; -- Ortalama siparis degeri

  2. MIN(sutun): Bir sutundaki minimum NULL olmayan degeri bulur (sayisal, metin, tarih turleri icin calisir).

    -- Standart SQL Ornekleri
SELECT MIN(order_total) FROM orders WHERE status = "completed"; -- En erken siparis tarihi

  3. MAX(sutun): Bir sutundaki maksimum NULL olmayan degeri bulur. 

    -- Standart SQL Ornekleri
SELECT MAX(order_total) FROM orders; -- En yuksek siparis toplami

Yaygin Toplama Fonksiyonu Uzantilari (Soz Dizimi/Kullanilabilirlik Degisir):

Bu fonksiyonlar yaygin olarak desteklenir ancak veritabani sistemleri arasinda farkli adlara veya soz dizimine sahip olabilir.

  • Metin Birlestirme: Birden fazla satirdaki metinleri tek bir metinde birlestirir.
    • PostgreSQL/Db2/Oracle (11g+): STRING_AGG(ifade, ayirici [ORDER BY ...])
    • MySQL/SQLite: GROUP_CONCAT(ifade [ORDER BY ...] [SEPARATOR ayirici])
    • SQL Server (2017+): STRING_AGG(ifade, ayirici) WITHIN GROUP (ORDER BY ...) 
    -- PostgreSQL Ornegi: Bir kategorideki urunlerin virgule ayrilmis listesi
SELECT category_name, STRING_AGG(product_name, ", " ORDER BY product_name)
FROM products
GROUP BY category_name;
  • Dizi Birlestirme: Degerleri bir dizide toplar (daha az yaygin, PostgreSQL gibi sistemlere ozgu).
    • PostgreSQL: ARRAY_AGG(ifade [ORDER BY ...])
    -- PostgreSQL Ornegi: Kategori basina urun ID'lerinin dizisi
SELECT category_id, ARRAY_AGG(product_id ORDER BY product_id)
FROM products
GROUP BY category_id;
  • Istatistiksel Fonksiyonlar: Standart sapma (STDDEV_SAMP, STDDEV_POP), varyans (VAR_SAMP, VAR_POP), korelasyon (CORR), vb. Kullanilabilirlik ve tam isimler degisebilir.

NULL Degerlerinin Islenmesi: Standart toplama fonksiyonlari (SUM, AVG, MIN, MAX) NULL degerlerini yok sayar. COUNT(*) tum satirlari sayar, COUNT(sutun) ise o sutundaki yalnizca NULL olmayan satirlari sayar.

GROUP BY ile Verileri Gruplama (Standart SQL)

GROUP BY cumlesini, belirtilen bir veya daha fazla sutunda ayni degerlere sahip satirlari bir ozet satirinda gruplar. Toplama fonksiyonlari daha sonra her gruba bagimsiz olarak uygulanir.

Standart SQL Soz Dizimi:

SELECT 
    gruplama_sutunu1, 
    gruplama_sutunu2, ..., 
    toplama_fonksiyonu(sutun)
FROM 
    tablo_adi
WHERE 
    filtre_kosulu -- Istege bagli: Gruplamadan ONCE satirlari filtreler
GROUP BY 
    gruplama_sutunu1, 
    gruplama_sutunu2, ...
ORDER BY 
    ...; -- Istege bagli: Gruplanmis sonuclari siralar

Onemli Kural (Standart SQL): SELECT listesindeki herhangi bir toplanmamis sutun mutlaka GROUP BY cumlesinde yer almalidir. (Eski MySQL surumlerinin gevsetilmis bir varsayilan modu vardi, ancak buna guvenemk standart degil ve genellikle belirsiz sonuclara yol acar).

Ornekler:

-- Standart SQL: Musteri basina siparis sayisini say
SELECT 
    customer_id, 
    COUNT(*) AS siparis_sayisi
FROM 
    orders
GROUP BY 
    customer_id;

Ornekler:

-- Standart SQL: Urun kategorisi basina toplam satis tutari
SELECT 
    p.category_id, 
    c.category_name, -- category_name SELECT'te oldugu icin GROUP BY'a dahil edin
    SUM(oi.quantity * oi.unit_price) AS toplam_kategori_satislari
FROM 
    order_items oi
JOIN 
    products p ON oi.product_id = p.product_id
JOIN 
    categories c ON p.category_id = c.category_id
GROUP BY 
    p.category_id, c.category_name -- Hem ID hem de ada gore grupla
ORDER BY 
    toplam_kategori_satislari DESC;

Ornekler:

-- Standart SQL: Tarih parcasina gore gruplama (fonksiyon soz dizimi biraz degisebilir)
SELECT 
    EXTRACT(YEAR FROM order_date) AS siparis_yili, -- Standart fonksiyon
    COUNT(*) AS yillik_siparis_sayisi
FROM 
    orders
GROUP BY 
    EXTRACT(YEAR FROM order_date)
ORDER BY 
    siparis_yili;
-- Not: DATE_TRUNC (PostgreSQL) veya satici-ozgu tarih bicimlendirme fonksiyonlari gibi fonksiyonlar burada sikca kullanilir.

Gelismis Gruplama: GROUPING SETS, ROLLUP, CUBE (Standart SQL Uzantilari):

Bu cumleler daha karmasik gruplama yetenekleri saglar ve tek bir sorguda birden fazla gruplama duzeyine veya kombinasyonuna izin verir. Daha sonraki SQL standartlarinin bir parcasidir (SQL:1999/SQL:2003) ve cogu buyuk modern veritabani (PostgreSQL, Oracle, SQL Server, Db2) tarafindan desteklenir, ancak soz dizimi detaylari veya sinirlamalar olabilir.

  1. GROUPING SETS: Birden fazla bagimsiz GROUP BY cumlesi belirtmeye izin verir. 
    -- Ornek: (Urun)'e, (Bolge)'ye ve genel toplama gore satis toplamlari
SELECT product, region, SUM(sales) AS toplam_satislar
FROM sales_data
GROUP BY GROUPING SETS ((product),(region));
  2. ROLLUP: Hiyerarsik ozetler icin gruplamalar olusturur (ornegin, Yil -> Ceyrek -> Ay, Ulke -> Bolge -> Sehir). 
    -- Ornek: (Urun)'e, (Bolge)'ye ve genel toplama gore satis toplamlari
SELECT year, quarter, month, SUM(sales) AS toplam_satislar
FROM sales_data
GROUP BY ROLLUP(year, quarter, month);
  3. CUBE: Belirtilen sutunlarin tum olasi kombinasyonlari icin gruplamalar olusturur. 
    -- Ornek: (Urun)'e, (Bolge)'ye ve genel toplama gore satis toplamlari
SELECT product, region, SUM(sales) AS toplam_satislar
FROM sales_data
GROUP BY CUBE(product, region);
    Bunlar veri ambarlama ve OLAP tarzindaki sorgular icin guclu araclardir ancak daha ileri duzey konulardir.

HAVING ile Gruplari Filtreleme (Standart SQL)

HAVING cumlesi bir GROUP BY sorgusunun sonuclarini filtreler. Kosullari genellikle toplama fonksiyonlari iceren gruplarin kendilerine uygular.

Standart SQL Soz Dizimi:

SELECT 
    gruplama_sutunu1, 
    toplama_fonksiyonu(sutun)
FROM 
    tablo_adi
WHERE 
    satir_filtre_kosulu -- Istege bagli: Gruplamadan ONCE satirlari filtreler
GROUP BY 
    gruplama_sutunu1
HAVING 
    grup_filtre_kosulu; -- Gruplamadan SONRA gruplari filtreler

WHERE ve HAVING Karsilastirmasi: - WHERE bireysel satirlari GROUP BY tarafindan islenmeden once filtreler. - HAVING GROUP BY tarafindan olusturulduktan ve toplamalar hesaplandiktan sonra tum gruplari filtreler. - HAVING icindeki kosullar genellikle toplama fonksiyonlari icerir; WHERE icindeki kosullar iceremez (cunku toplamalar henuz olusturulmamis gruplar uzerinde calisir).

Ornekler:

-- Standart SQL: 5'ten fazla siparisi olan musterileri bul
SELECT 
    customer_id, 
    COUNT(*) AS siparis_sayisi
FROM 
    orders
GROUP BY 
    customer_id
HAVING 
    COUNT(*) > 5; -- Toplama COUNT'a gore gruplari filtrele

Ornekler:

-- Standart SQL: Ortalama urun fiyati 100$'in uzerinde olan urun kategorilerini bul
SELECT 
    p.category_id, 
    c.category_name,
    AVG(p.list_price) AS ortalama_fiyat
FROM 
    products p
JOIN 
    categories c ON p.category_id = c.category_id
GROUP BY 
    p.category_id, c.category_name
HAVING 
    AVG(p.list_price) > 100;

Ornekler:

-- Standart SQL: WHERE ve HAVING'i birlestirme
-- 2023'te 3'ten fazla siparis veren 'USA'daki musterileri bul
SELECT 
    o.customer_id,
    c.customer_name,
    COUNT(*) AS siparis_sayisi_2023
FROM 
    orders o
JOIN 
    customers c ON o.customer_id = c.customer_id
WHERE 
    c.country = 'USA' 
    AND EXTRACT(YEAR FROM o.order_date) = 2023 -- Satirlari once filtrele
GROUP BY 
    o.customer_id, c.customer_name
HAVING 
    COUNT(*) > 3; -- Saymadan sonra gruplari filtrele

Pencere Fonksiyonlarina Giris (Standart SQL)

Pencere fonksiyonlari, mevcut satirla bir sekilde iliskili olan bir tablo satir kumesi ("pencere" olarak adlandirilir) uzerinde hesaplamalar yapar. GROUP BY ile birlikte toplama fonksiyonlarindan farkli olarak, pencere fonksiyonlari satirlari daraltmaz; pencereye dayali olarak her satir icin bir deger dondurur.

Pencere fonksiyonlari standart SQL'in bir parcasidir (SQL:2003 ve sonrasi) ve cogu modern RDBMS (PostgreSQL, Oracle, SQL Server, Db2, MySQL 8+, SQLite 3.25+) tarafindan desteklenir, ancak belirli fonksiyon kullanilabilirligi veya soz dizimi detaylari biraz degisebilir.

Standart SQL Soz Dizimi:

pencere_fonksiyonu(argumanlar) OVER (
    [PARTITION BY bolumleme_sutunu1, bolumleme_sutunu2, ...] -- Istege bagli: Satirlari bolumlere (gruplara) ayirir
    [ORDER BY siralama_sutunu1 [ASC|DESC], siralama_sutunu2 [ASC|DESC], ...] -- Istege bagli: Her bolum icindeki satirlari siralar
    [cerceve_cumlesi] -- Istege bagli: Bolum icindeki satirlarin alt kumesini tanimlar (pencere cercevesi)
)

Temel Kavramlar: - OVER() Cumlesi: Bir pencere fonksiyonunu belirten zorunlu cumle. - PARTITION BY: Satirlari bagimsiz bolumlere ayirir. Pencere fonksiyonu her bolume ayri ayri uygulanir. Atlanirsa, tum sonuc kumesi tek bir bolum olarak kabul edilir. - ORDER BY: Her bolum icindeki satirlarin sirasini belirler. Siralama ve sira bagli fonksiyonlar (LAG, LEAD, kumulatif toplamlar) icin cok onemlidir. - cerceve_cumlesi: Fonksiyonun hesaplamasina dahil edilecek, bolum icindeki mevcut satira gore satirlarin kumesini belirtir (ornegin, ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING). Atlanirsa, varsayilan cerceve genellikle fonksiyona ve ORDER BY varligina baglidir.

Yaygin Pencere Fonksiyonu Turleri (Cogunlukla Standart SQL):

  1. Siralama Fonksiyonlari:

    • ROW_NUMBER(): Bolumu icindeki her satira benzersiz bir ardisik tamsayi atar (ornegin, 1, 2, 3, 4).
    • RANK(): ORDER BY cumlesine gore siralama atar. Esitliklerden sonra siralamalari atlar (ornegin, 1, 2, 2, 4).
    • DENSE_RANK(): Esitliklerden sonra bosluk olmadan siralama atar (ornegin, 1, 2, 2, 3).
    • NTILE(n): Bolumu n kabaca esit gruba (kova) boler ve bir kova numarasi (1 ile n arasi) atar.
    -- Standart SQL: Calisanlari her departman icinde maasa gore sirala
SELECT 
    employee_name, 
    department_name, 
    salary,
    RANK() OVER (PARTITION BY department_name ORDER BY salary DESC) AS dept_maas_sirasi
FROM employees;

Yaygin Pencere Fonksiyonu Turleri (Cogunlukla Standart SQL):

  1. Deger Fonksiyonlari (Offset Fonksiyonlari):

    • LAG(ifade, [offset], [varsayilan]): ORDER BY'a gore bolum icindeki onceki bir satirdaki veriye erisir.
    • LEAD(ifade, [offset], [varsayilan]): Sonraki bir satirdaki veriye erisir.
    • FIRST_VALUE(ifade): Pencere cercevesindeki ilk satirdaki ifadenin degerini alir.
    • LAST_VALUE(ifade): Pencere cercevesindeki son satirdaki degeri alir.
    -- Standart SQL: Onceki aya kiyasla satis buyumesini hesapla
SELECT 
    sales_month, 
    monthly_total,
    LAG(monthly_total, 1, 0) OVER (ORDER BY sales_month) AS onceki_ay_toplami
FROM monthly_sales_summary;

Yaygin Pencere Fonksiyonu Turleri (Cogunlukla Standart SQL):

  1. Pencere Fonksiyonlari Olarak Toplama Fonksiyonlari: Standart toplama fonksiyonlari (SUM, AVG, COUNT, MIN, MAX) OVER() cumlesi eklenerek pencere fonksiyonlari olarak kullanilabilir. Her satir icin pencere cercevesi uzerinde calisirlar. 

    -- Standart SQL: Zaman icerisinde satislarin kumulatif toplamini hesapla
SELECT 
    order_date, 
    order_total,
    SUM(order_total) OVER (ORDER BY order_date ROWS UNBOUNDED PRECEDING) AS kumulatif_toplam_satislar
FROM orders;

-- Standart SQL: Her siiparisin toplam satislardaki yuzdesini hesapla
SELECT 
    order_id, 
    order_total,
    SUM(order_total) OVER () AS genel_toplam_satislar, -- Pencere tum sonuc kumesi
    (order_total / SUM(order_total) OVER ()) * 100 AS toplam_yuzde
FROM orders;

Pencere Cercevesi Cumlesi (Standart SQL): Mevcut satira gore hesaplama icin kullanilan bolum icindeki satirlarin alt kumesini tanimlar. Yaygin secenekler: - ROWS UNBOUNDED PRECEDING: Bolumun basindan mevcut satira kadar. - ROWS BETWEEN <n> PRECEDING AND <m> FOLLOWING: Mevcut satir etrafinda belirli bir aralik. - RANGE BETWEEN ...: Mevcut satirin ORDER BY degerine gore deger aralik (daha karmasik).

-- Standart SQL: Satislarin 3 gunluk hareketli ortalamasi
SELECT 
    sales_date,
    daily_total,
    AVG(daily_total) OVER (ORDER BY sales_date ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) AS hareketli_ort_3gun
FROM daily_sales;

Pratik Uygulamalar: Pencere fonksiyonlari su durumlar icin son derece gucludur: - Kumulatif toplamlar, hareketli ortalamalar hesaplama. - Gruplar icinde sonuclari siralama (En-N sorgular). - Satirlari onceki veya sonraki satirlarla karsilastirma (farklari, buyume oranlarini hesaplama). - Toplam yuzdeleri hesaplama.

Toplama fonksiyonlarini, GROUP BY, HAVING ve pencere fonksiyonlarini ustaca kullanmak, karmasik veri analizi ve raporlamayi dogrudan veritabaninda yapma yeteneginizi onemli olcude arttirir.
