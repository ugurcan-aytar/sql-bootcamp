Gun 1: Temel SQL Sorgulari

Temel SQL Sorgulari

SELECT Ifadeleri ve Temel Soz Dizimi

SELECT ifadesi, veritabani tablolarindan veri almak icin kullanilan SQL'in temel tasidir. Soz dizimini iyi bilmek, herhangi bir iliskisel veritabanini sorgulamak icin onemlidir.

Temel SELECT Soz Dizimi (Standart SQL):

SELECT column1, column2, ...
FROM table_name;

Bu, belirtilen tablodan belirli sutunlari getirir.

Tum Sutunlari Secme (SELECT *):

SELECT * FROM table_name;

Hizli kesfetme icin kullanisli olsa da (SELECT * standart SQL'dir), uygulama kodunda kullanilmasi tum veritabani sistemlerinde genellikle onerilmez cunku: - Gerekenden fazla veri cekebilir, ag trafigini ve performansi etkiler. - Hangi sutunlarin gercekten kullanildigini gizledigi icin kodu daha az okunabilir kilar. - Tablo yapisi degisirse (ornegin, sutunlar eklenirse/kaldirilirsa) uygulamalari bozabilir.

Belirli Sutunlari Secme: Her zaman ihtiyaciniz olan belirli sutunlari listelemeyi tercih edin:

-- Example using Odoo schema (PostgreSQL)
SELECT name, email, phone
FROM res_partner;

Sutun Takma Adlari (Standart SQL): Takma adlar, sorgu sonucundaki sutunlar veya ifadeler icin gecici ve genellikle daha okunabilir adlar saglar. AS anahtar kelimesi standarttir ancak genellikle istege baglidir.

SELECT 
    name AS "Customer Name", -- Quoted alias preserves case/spaces (standard)
    email AS EmailAddress,   -- Unquoted alias (case usually ignored/lowercased)
    phone MobilePhone        -- AS keyword is optional
FROM res_partner;
  • Lehce Notu: Takma adlar icin cift tirnak standart SQL olsa da, bazi sistemler farkli tirnaklama kullanir (ornegin, MySQL'de ters tirnak `, SQL Server'da koseli parantez []). Tirnaksiz takma adlar genellikle buyuk/kucuk harf duyarsizdir.

SELECT'te Ifadeler (Standart SQL): SELECT listesinde dogrudan sutunlar uzerinde hesaplamalar yapabilir veya fonksiyonlar kullanabilirsiniz.

-- Example using Odoo schema (PostgreSQL)
SELECT 
    product_id,
    price_unit,
    product_uom_qty, -- Odoo often uses product_uom_qty for quantity
    price_unit * product_uom_qty AS line_total
FROM sale_order_line;

DISTINCT Anahtar Kelimesi (Standart SQL): DISTINCT, sonuc kumesinden yinelenen satirlari kaldirır. Secilen sutunlara gore tum satiri dikkate alir.

-- Get unique country IDs from partners
SELECT DISTINCT country_id
FROM res_partner;

DISTINCT ON (PostgreSQL Uzantisi): PostgreSQL, ORDER BY maddesine gore ifadenin her benzersiz degeri icin ilk satiri donduren DISTINCT ON (expression) sunar. Bu cok kullanislidir ancak standart SQL degildir.

-- Get one partner per country (the most recently created one)
-- PostgreSQL specific syntax!
SELECT DISTINCT ON (country_id) country_id, id, name
FROM res_partner
ORDER BY country_id, create_date DESC;
  • Standart Alternatif: Standart SQL'de ayni sonucu elde etmek genellikle daha sonra ele alacagimiz ROW_NUMBER() gibi pencere fonksiyonlarini icerir.

WHERE Maddesi ile Filtreleme (Standart SQL)

WHERE maddesi, belirtilen kosullara gore herhangi bir gruplama veya toplama isleminden once satirlari filtreler.

Temel Soz Dizimi:

SELECT column_list
FROM table_name
WHERE condition;

Karsilastirma Operatorleri (Standart SQL): - Esit: = - Esit degil: <> (standart), != (yaygin uzanti) - Buyuktur: > - Kucuktur: < - Buyuk veya esit: >= - Kucuk veya esit: <=

Ornekler:

-- Partners from the United States (assuming country_id 233 is US)
SELECT name, email, city
FROM res_partner
WHERE country_id = 233;

-- Products with a list price greater than 100
SELECT name, list_price
FROM product_template
WHERE list_price > 100;

-- Orders placed on or after a specific date
SELECT name, date_order
FROM sale_order
WHERE date_order >= '2023-01-01'; -- Date format 'YYYY-MM-DD' is standard

Mantiksal Operatorler (Standart SQL): - AND: Her iki kosul da dogru olmalidir. - OR: Kosullardan en az biri dogru olmalidir. - NOT: Bir kosulun dogru degerini tersine cevirir. Karmasik kosullarda degerlendirme sirasini kontrol etmek icin parantez () kullanin.

Ornekler:

-- Partners from the US OR Canada (IDs 233, 20)
SELECT name, country_id
FROM res_partner
WHERE country_id = 233 OR country_id = 20;

-- Partners from the US OR Canada (IDs 233, 20)
SELECT name, country_id
FROM res_partner
WHERE country_id IN (233, 20);

-- Active products that are in stock
SELECT name, sq.quantity
FROM product_template pt
LEFT JOIN product_product pp on pt.id = pp.product_tmpl_id
LEFT JOIN stock_quant sq on pp.id = sq.product_id
WHERE pt.active = true AND sq.quantity > 0;

-- Orders that are NOT cancelled
SELECT name, state
FROM sale_order
WHERE NOT state = 'cancel';
-- Equivalent using <> or !=
-- WHERE state <> 'cancel';

NULL Degerlerini Isleme (Standart SQL): NULL bilinmeyen veya eksik bir degeri temsil eder. = veya <> gibi standart operatorlerle karsilastirilamaz. Bunun yerine IS NULL veya IS NOT NULL kullanin. 

-- Partners missing an email address
SELECT name
FROM res_partner
WHERE email IS NULL;

-- Partners who have an email address
SELECT name, email
FROM res_partner
WHERE email IS NOT NULL;
  • Onemli: WHERE email = NULL veya WHERE email <> NULL standart SQL'de beklenildigi gibi calismaz cunku NULL kendisi dahil hicbir seye esit (veya esit olmayan) degildir.

BETWEEN Operatoru (Standart SQL): Bir degerin belirtilen aralik icinde (dahil) olup olmadigini kontrol eder. 

SELECT name, list_price
FROM product_template
WHERE list_price BETWEEN 100 AND 500; -- Equivalent to: list_price >= 100 AND list_price <= 500

IN Operatoru (Standart SQL): Bir degerin saglanan listedeki herhangi bir degerle eslesip eslesmedigini kontrol eder.

SELECT name, country_id
FROM res_partner
WHERE country_id IN (235, 38, 73); -- US, Canada, Germany

Bu, genellikle birden fazla OR kosulundan daha okunabilirdir.

LIKE Operatoru (Standart SQL): Metin verilerinde basit desen eslestirme yapar. - %: Sifir veya daha fazla karakterden olusan herhangi bir diziyle eslesir. - _: Tam olarak bir karakterle eslesir. 

-- Partner names starting with 'Geo'
SELECT name
FROM res_partner
WHERE name LIKE 'Geo%';

-- Emails ending with '.org'
SELECT name, email
FROM res_partner
WHERE email LIKE '%.org';

-- Product codes with 'X' as the second character (e.g., 'AX123')
SELECT default_code
FROM product_template
WHERE default_code LIKE '_-%';
  • Buyuk/Kucuk Harf Duyarliligi: Standart SQL LIKE buyuk/kucuk harf duyarlidir. Ancak, varsayilan davranis veritabanlari arasinda degisir ve harmanlama ayarlarina baglidir (ornegin, SQL Server ve MySQL'de genellikle varsayilan olarak buyuk/kucuk harf duyarsiz, PostgreSQL ve Oracle'da buyuk/kucuk harf duyarli).
  • Buyuk/Kucuk Harf Duyarsiz LIKE:
    • PostgreSQL: ILIKE kullanin (ornegin, WHERE name ILIKE 'geo%').
    • Diger Veritabanlari: Genellikle her iki tarafta LOWER() veya UPPER() gibi fonksiyonlar kullanilarak elde edilir (ornegin, WHERE LOWER(name) LIKE 'geo%') veya buyuk/kucuk harf duyarsiz harmanlamalar ayarlanarak.

Duzensiz Ifadeler (Standart SQL Degil, Soz Dizimi Degisir): Cogu veritabani duzensiz ifadeler kullanarak daha guclu desen eslestirmeyi destekler, ancak soz dizimi onemli olcude degisir.

-- PostgreSQL Example: Names containing numbers
SELECT name, email FROM res_partner WHERE email ~ '[0-9]';

-- PostgreSQL Example: Case-insensitive regex for names starting with 'a'
SELECT name FROM res_partner WHERE name ~* '^a'; 

-- MySQL Example: Using REGEXP or RLIKE
-- SELECT name FROM res_partner WHERE name REGEXP '[0-9]';

-- Oracle Example: Using REGEXP_LIKE
-- SELECT name FROM res_partner WHERE REGEXP_LIKE(name, '[0-9]');

-- SQL Server Example: Using PATINDEX (limited regex)
-- SELECT name FROM res_partner WHERE PATINDEX('%[0-9]%', name) > 0;

ORDER BY ile Siralama (Standart SQL)

ORDER BY maddesi, sonuc kumesindeki satirlari siralar.

Temel Soz Dizimi:

SELECT column_list
FROM table_name
WHERE condition -- Optional
ORDER BY column1 [ASC|DESC], column2 [ASC|DESC], ...;

Siralama Yonu: - ASC: Artan sirada (A-Z, 0-9). Belirtilmezse bu varsayilandir. - DESC: Azalan sirada (Z-A, 9-0).

Ornekler:

-- Sort partners alphabetically by name
SELECT name, city
FROM res_partner
ORDER BY name ASC; -- ASC is optional here

-- Sort products by price, highest first
SELECT name, list_price
FROM product_template
ORDER BY list_price DESC;

-- Sort partners by country, then by city within each country
SELECT name, country_id, city
FROM res_partner
ORDER BY country_id ASC, city ASC;

-- Sort by an expression (e.g., length of name)
SELECT name
FROM res_partner
ORDER BY LENGTH(name) DESC; -- Function name might vary (e.g., LEN in SQL Server)

Siralamada NULL Degerlerini Isleme (Davranis Degisir): ASC veya DESC kullanildiginda NULL degerlerinin nasil siralandigi (FIRST veya LAST) veritabani sistemine baglidir. - PostgreSQL/Oracle: ASC icin varsayilan NULLS LAST, DESC icin NULLS FIRST. - SQL Server/MySQL: ASC icin varsayilan NULLS FIRST, DESC icin NULLS LAST. - Acik Kontrol (PostgreSQL/Oracle): NULLS FIRST veya NULLS LAST'i acikca belirtebilirsiniz. sql -- PostgreSQL/Oracle syntax SELECT name, email FROM res_partner ORDER BY email ASC NULLS FIRST; SELECT name, email FROM res_partner ORDER BY email DESC NULLS LAST;

Sonuclari Sinirlandirma (Soz Dizimi Degisir)

Dondurulecek satir sayisini kisitlamak, ozellikle sayfalama icin yaygindir. Soz dizimi tarihsel olarak degismistir ancak SQL standardina dogru yaklasmakladir.

PostgreSQL / MySQL Soz Dizimi:

SELECT column_list
FROM table_name
ORDER BY sort_column -- ORDER BY is crucial for predictable LIMIT
LIMIT row_count [OFFSET starting_row];
  • LIMIT row_count: Dondurulebilecek maksimum satir sayisini belirtir.
  • OFFSET starting_row: Satirlari dondurmeye baslamadan once atlanacak satir sayisini belirtir (0 tabanli).

Ornekler (PostgreSQL/MySQL):

-- Get the top 10 most recently created partners
SELECT name, create_date
FROM res_partner
ORDER BY create_date DESC
LIMIT 10;

-- Get the third page of 25 partners (rows 51-75)
SELECT name
FROM res_partner
ORDER BY name
LIMIT 25 OFFSET 25; -- Skip 25 rows (1 page * 25 rows/page)

Standart SQL Soz Dizimi (PostgreSQL 8.4+, Oracle 12c+, SQL Server 2012+, Db2):

SELECT column_list
FROM table_name
ORDER BY sort_column
OFFSET starting_row ROWS
FETCH FIRST row_count ROWS ONLY; -- or FETCH NEXT ...

Ornek (Standart SQL):

-- Get the top 10 most recently created partners
SELECT name, create_date
FROM res_partner
ORDER BY create_date DESC
FETCH FIRST 10 ROWS ONLY;

-- Get the second page of 25 partners (rows 26-50)
SELECT name
FROM res_partner
ORDER BY name
OFFSET 25 ROWS FETCH NEXT 25 ROWS ONLY;

Eski Satici-Spesifik Soz Dizimi: - SQL Server (Eski): SELECT TOP row_count column_list ... ORDER BY ... (Kolay OFFSET yok) - Oracle (Eski): SELECT column_list ... WHERE ROWNUM <= row_count ... ORDER BY ... (OFFSET icin karmasik)

Pratik Ornekler (Odoo/PostgreSQL Semasini Kullanarak)

Bu ornekler PostgreSQL soz dizimini kullanir ancak birçok is veritabanina uygulanabilir yaygin veri alma gorevlerini gosterir.

Ornek 1: Son, yuksek degerli satis siparislerini bulma

SELECT 
    so.name AS order_ref,
    rp.name AS customer_name,
    so.date_order,
    so.amount_total
FROM 
    sale_order so
JOIN 
    res_partner rp ON so.partner_id = rp.id
WHERE 
    so.state IN ('sale', 'done') -- Consider only confirmed/done orders
    AND so.date_order >= CURRENT_DATE - INTERVAL '90 days' -- Orders in the last 90 days
    AND so.amount_total > 1000 -- Orders over $1000
ORDER BY 
    so.amount_total DESC
LIMIT 20;
  • Not: CURRENT_DATE ve INTERVAL soz dizimi diger veritabanlarinda biraz farkli olabilir.

Ornek 2: Stoku dusuk urunleri bulma

SELECT 
    pt.name AS product_name,
    pt.default_code AS internal_reference,
    SUM(sq.quantity) AS quantity_on_hand -- Assumes stock.quant stores current levels
FROM 
    product_template pt
JOIN 
    product_product pp ON pp.product_tmpl_id = pt.id
LEFT JOIN -- Use LEFT JOIN in case a product has no stock record yet
    stock_quant sq ON sq.product_id = pp.id -- Need to consider location too in real scenario
WHERE 
    pt.active = true
    AND pt.type = 'consu' -- Filter for storable products
GROUP BY -- Need GROUP BY when using SUM()
    pt.id, pt.name, pt.default_code
HAVING -- Filter *after* aggregation
    SUM(sq.quantity) < 10 -- Assuming reorder point is 10
ORDER BY 
    quantity_on_hand ASC;
  • Not: GROUP BY ve HAVING daha sonra daha ayrintili olarak ele alinacaktir. Envanter seviyelerini sorgulama kavrami sistemler arasinda yaygindir.

Ornek 3: Belirli e-posta alanli musterileri bulma

SELECT 
    name,
    email
FROM 
    res_partner
WHERE 
    email IS NOT NULL 
    AND (
        email LIKE '%@example.com' 
        OR email LIKE '%@sample.org'
    )
ORDER BY 
    name
LIMIT 50;
  • Not: Bu standart LIKE kullanir ve buyuk/kucuk harf duyarliligi gereksinimlere uygunsa veya uygun fonksiyonlar/harmanlamalar kullaniliyorsa cogu veritabaninda calismalidir.

Bu temel SELECT ifade bilesenlerini (SELECT, FROM, WHERE, ORDER BY, LIMIT/FETCH) anlamak SQL ogrenmenin ilk buyuk adimidir. Herhangi bir iliskisel veritabanindan veri almaniza, filtrelemenize ve siralamaniza olanak tanirlar.
