Giris: Buyuk Iliskisel Veritabani Sistemlerinin Karsilastirilmasi

Genel Bakis

Bu egitim uygulamali calismalar icin oncelikli olarak PostgreSQL'e odaklansa da, Iliskisel Veritabani Yonetim Sistemleri (RDBMS) hakkinda genis bir perspektife sahip olmak, urun sahibi veya veri profesyoneli olmak isteyen herkes icin cok onemlidir. SQL (Yapilandirilmis Sorgu Dili), iliskisel veritabanlariyla etkilesim icin standart dildir, ancak her RDBMS'nin kendi gecmisi, guclui yonleri, zayif yonleri ve SQL sozdiziminde kucuk farkliliklari (genellikle SQL lehceleri olarak adlandirilir) vardir.

Bu bolum, en populer RDBMS'lerden bazilari hakkinda ust duzey bir karsilastirma sunmaktadir: PostgreSQL, MySQL, Oracle Database, Microsoft SQL Server ve IBM Db2.

SQL Lehce Farkliliklari (Ornekler)

SQL sozdiziminin degisebilecegini anlamak onemlidir. Iste birkac yaygin ornek:

  1. Otomatik Artan Sutunlar:
    • PostgreSQL: SERIAL veya GENERATED ALWAYS AS IDENTITY
    • MySQL: AUTO_INCREMENT
    • SQL Server: IDENTITY(1,1)
    • Oracle: GENERATED AS IDENTITY veya Sequences
    • Db2: GENERATED ALWAYS AS IDENTITY
  2. Sonuclari Sinirlandirma:
    • PostgreSQL/MySQL: LIMIT <sayi> OFFSET <baslangic>
    • SQL Server: OFFSET <baslangic> ROWS FETCH NEXT <sayi> ROWS ONLY (SQL 2012+) veya TOP <sayi> (eski surumler)
    • Oracle: FETCH FIRST <sayi> ROWS ONLY (12c+) veya ROWNUM <= <sayi> (eski surumler)
    • Db2: FETCH FIRST <sayi> ROWS ONLY
  1. String Birlestirme:
    • PostgreSQL/Oracle/Db2: || operatoru
    • MySQL: CONCAT() fonksiyonu
    • SQL Server: + operatoru veya CONCAT() fonksiyonu
  2. Tarih/Zaman Fonksiyonlari:
    • PostgreSQL: NOW(), CURRENT_DATE, EXTRACT(EPOCH FROM ...)
    • MySQL: NOW(), CURDATE(), UNIX_TIMESTAMP()
    • SQL Server: GETDATE(), CAST(GETDATE() AS DATE), DATEDIFF(s, ...)
    • Oracle: SYSDATE, TRUNC(SYSDATE), (date1 - date2) * 86400
    • Db2: CURRENT TIMESTAMP, CURRENT DATE
  3. Sorgularda Kosullu Mantik:
    • Standart SQL (Tumu): CASE WHEN ... THEN ... ELSE ... END
    • PostgreSQL/Oracle/Db2: DECODE() (Oracle/Db2'ye ozgu, artik daha az yaygin)
    • MySQL: IF(kosul, dogru_deger, yanlis_deger), IFNULL(deger1, deger2)
    • SQL Server: IIF(kosul, dogru_deger, yanlis_deger), ISNULL(deger1, deger2)
    • PostgreSQL: COALESCE(deger1, deger2, ...) (Standart), NULLIF(deger1, deger2) (Standart)

Neden Burada PostgreSQL'e Odaklaniyoruz?

Diger RDBMS'lerin onemini kabul etmekle birlikte, bu egitim uygulamali calisma icin PostgreSQL kullanmaktadir, cunku:

  • Acik Kaynak Yapisi: Ogrenme ve gelistirme icin ucretsiz olarak mevcuttur.
  • SQL Uyumlulugu: SQL standartlarina yakindan uyar, bu da ogrenilen kavramlari kolayca aktariabilir kilar.
  • Zengin Ozellik Seti: Modern uygulamalarda ilgili olan JSONB, pencere fonksiyonlari ve genisletilebilirlik gibi gelismis ozellikleri destekler.
  • Artan Popularite: Yeni projeler icin giderek daha fazla tercih edilmektedir.
  • Ozel Ortaminiz: Planlanmis Neon veritabani kurulumu ile uyumludur.

Kurs boyunca, ilgili oldugu yerlerde diger buyuk veritabanlarina kiyasla sozdizimi veya davranistaki onemli farkliliklara dikkat cekecegiz, boylece cesitli platformlarda uygulanabilir SQL ilkeleri hakkinda genis bir anlayis kazanmanizi saglayacagiz.
