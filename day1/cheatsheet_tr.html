Bölüm 1: SQL Temelleri

Veritabanlari (Databases), SELECT, JOIN'ler & Gelismis Sorgular (Queries)

Hizli Referans - SQL Sorgu Sirasi (Query Order)
SELECT sutunlar      -- 5. Sutunlari sec (Select columns)
FROM tablo           -- 1. Tabloyu sec (Choose table)
JOIN tablo2 ON ...  -- 2. Tablolari birlestir (Join tables)
WHERE kosul          -- 3. Satirlari filtrele (Filter rows)
GROUP BY sutun       -- 4. Satirlari grupla (Group rows)
HAVING kosul         -- 6. Gruplari filtrele (Filter groups)
ORDER BY sutun       -- 7. Sonuclari sirala (Sort results)
LIMIT n              -- 8. Satir sayisini sinirla (Limit rows)

Veritabani Temelleri (Database Fundamentals)

Temel Kavramlar (Basic Concepts)
  • Veritabani (Database): Yapilandirilmis verilerin organize koleksiyonu
  • Tablo (Table): Satir ve sutunlarda iliskili veri koleksiyonu
  • Satir (Row/Record): Tablodaki tek bir kayit
  • Sutun (Column/Field): Verinin bir ozelligi
  • Sema (Schema): Veritabani yapisinin semasi
Anahtarlar & Kisitlamalar (Keys & Constraints)
  • Birincil Anahtar (PRIMARY KEY): Her satir icin benzersiz tanimlayici
  • Yabanci Anahtar (FOREIGN KEY): Baska tablonun primary key'ine baglanti
  • Benzersiz (UNIQUE): Tekrar eden degerlere izin verilmez
  • Bos Olamaz (NOT NULL): Deger mutlaka saglanmali
  • Kontrol (CHECK): Deger belirli kosulu karsilamali

SELECT Ifadesi (SELECT Statement)

Temel SELECT Sozdizimi (Basic Syntax)
-- Tum sutunlari sec (Select all columns)
SELECT * FROM employees;

-- Belirli sutunlari sec (Select specific columns)
SELECT first_name, last_name, salary
FROM employees;

-- Sutun takma adi (Column alias)
SELECT first_name AS "Ad",
       salary * 12 AS yillik_maas
FROM employees;
-- Tekrarlari kaldir (Remove duplicates)
SELECT DISTINCT department_id
FROM employees;

-- Sonuclari sinirla (Limit results)
SELECT * FROM employees
LIMIT 10;

-- Satirlari atla - sayfalama (Pagination)
SELECT * FROM employees
LIMIT 10 OFFSET 20;

WHERE ile Filtreleme (Filtering with WHERE)

Karsilastirma Operatorleri (Comparison Operators)
SELECT * FROM employees WHERE
  salary = 50000          -- Esit (Equal)
  salary != 50000         -- Esit degil (Not equal)
  salary > 50000          -- Buyuk (Greater than)
  salary >= 50000         -- Buyuk veya esit (Greater or equal)
  salary < 50000          -- Kucuk (Less than)
  salary <= 50000         -- Kucuk veya esit (Less or equal)
  salary BETWEEN 40000 AND 60000
  department_id IN (1, 2, 3)
  first_name LIKE 'J%'    -- J ile baslar (Starts with J)
  email IS NULL           -- NULL kontrolu (NULL check)
  email IS NOT NULL
Mantiksal Operatorler (Logical Operators)
-- AND: Her iki kosul da dogru olmali (Both conditions must be true)
SELECT * FROM employees
WHERE salary > 50000
  AND department_id = 5;

-- OR: En az bir kosul dogru olmali (At least one condition must be true)
SELECT * FROM employees
WHERE department_id = 1
  OR department_id = 2;

-- NOT: Kosulu olumsuzla (Negate condition)
SELECT * FROM employees
WHERE NOT department_id = 5;

-- Parantezlerle birlesik (Combined with parentheses)
WHERE (dept = 1 OR dept = 2)
  AND salary > 50000

JOIN Turleri (JOIN Types)
JOIN Turu (Type)Aciklama (Description)Dondurur (Returns)

INNER JOIN
-- Sadece eslesen satirlari dondurur (Returns only matching rows)
SELECT e.name, d.department_name
FROM employees e
INNER JOIN departments d
  ON e.department_id = d.id;
LEFT JOIN
-- Departmani olmasa bile tum calisanlar (All employees even without department)
SELECT e.name, d.department_name
FROM employees e
LEFT JOIN departments d
  ON e.department_id = d.id;
Coklu JOIN'ler (Multiple JOINs)
SELECT
    e.first_name,
    d.department_name,
    l.city
FROM employees e
INNER JOIN departments d ON e.department_id = d.department_id
INNER JOIN locations l ON d.location_id = l.location_id
WHERE l.country_id = 'US';

Toplama Fonksiyonlari (Aggregate Functions)

Yaygin Toplama Fonksiyonlari (Common Aggregate Functions)
COUNT(*)        -- Tum satirlari say (Count all rows)
COUNT(sutun)    -- NULL olmayan degerleri say (Count non-NULL)
SUM(sutun)      -- Degerlerin toplami (Sum of values)
AVG(sutun)      -- Ortalama deger (Average value)
MIN(sutun)      -- Minimum deger (Minimum value)
MAX(sutun)      -- Maksimum deger (Maximum value)
GROUP BY & HAVING (Gruplama ve Filtreleme)
SELECT department_id,
       COUNT(*) AS calisan_sayisi,
       AVG(salary) AS ort_maas
FROM employees
GROUP BY department_id
HAVING COUNT(*) > 5
ORDER BY ort_maas DESC;

Alt Sorgular (Subqueries)

Skaler Alt Sorgu - Tek Deger (Scalar Subquery - Single Value)
-- Ortalamadan fazla kazanan calisanlar (Employees earning above average)
SELECT * FROM employees
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
);
Liste Alt Sorgusu - Coklu Deger (List Subquery - Multiple Values)
-- Satis veya Pazarlamadaki calisanlar (Employees in Sales or Marketing)
SELECT * FROM employees
WHERE department_id IN (
    SELECT id FROM departments
    WHERE name IN ('Sales', 'Marketing')
);
EXISTS Alt Sorgusu (EXISTS Subquery)
-- Calisani olan departmanlar (Departments with employees)
SELECT * FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e
    WHERE e.department_id = d.id
);

Ortak Tablo Ifadeleri - CTE (Common Table Expressions)

WITH Cumleciegi (WITH Clause)
WITH dept_stats AS (
    SELECT department_id,
           AVG(salary) AS ort_maas,
           COUNT(*) AS calisan_sayisi
    FROM employees
    GROUP BY department_id
)
SELECT d.name, ds.ort_maas, ds.calisan_sayisi
FROM departments d
JOIN dept_stats ds ON d.id = ds.department_id
WHERE ds.ort_maas > 50000;
CTE'ler okunabilirligi arttirir ve ana sorguda birden fazla kez referans edilebilir. (CTEs improve readability and can be referenced multiple times in the main query.)

Pencere Fonksiyonlari (Window Functions)

Pencere Fonksiyonu Sozdizimi (Window Function Syntax)
SELECT
    employee_name,
    department_id,
    salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) AS siralama,
    RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) AS dept_sirasi,
    SUM(salary) OVER (PARTITION BY department_id) AS dept_toplam,
    AVG(salary) OVER () AS sirket_ort
FROM employees;
Yaygin Pencere Fonksiyonlari (Common Window Functions)
Fonksiyon (Function)Aciklama (Description)
ROW_NUMBER()Her satir icin benzersiz sira numarasi (Unique row number)
RANK()Esitliklerde bosluklu siralama (Ranking with gaps)
DENSE_RANK()Esitliklerde bosluksuz siralama (Ranking without gaps)
LAG(col, n)n satir onceki deger (Value n rows before)
LEAD(col, n)n satir sonraki deger (Value n rows after)
FIRST_VALUE(col)Penceredeki ilk deger (First value in window)
LAST_VALUE(col)Penceredeki son deger (Last value in window)

Sonuclari Siralama (Sorting Results)

ORDER BY Ornekleri (ORDER BY Examples)
-- Artan - varsayilan (Ascending - default)
SELECT * FROM employees ORDER BY last_name;

-- Azalan (Descending)
SELECT * FROM employees ORDER BY salary DESC;

-- Coklu sutun (Multiple columns)
SELECT * FROM employees
ORDER BY department_id ASC, salary DESC;

-- NULL'lari yonetme (Handling NULLs)
ORDER BY commission NULLS LAST;
ORDER BY commission NULLS FIRST;

En Iyi Uygulamalar (Best Practices)

Anlamli takma adlar (aliases) kullanin - Karmasik sorgulari okunabilir kilar
Sutunlari (columns) belirtin - Uretimde SELECT * kullanmayin
Tablo takma adlari (table aliases) kullanin - Self-join'lerde gerekli, genel olarak iyi aliskanlik
NULL, NULL'a esit degildir (NULL is not equal to NULL) - = NULL degil, IS NULL kullanin
GROUP BY tuzagi (GROUP BY trap) - Toplanmamis tum sutunlar GROUP BY'da olmali
ON olmadan JOIN (JOIN without ON) - Kartezyen carpim (Cartesian product) olusturur (genellikle hatali)
