Introduction: Comparing Major Relational Database Systems

Overview

While this training focuses primarily on PostgreSQL for hands-on exercises, understanding the broader landscape of Relational Database Management Systems (RDBMS) is crucial for any aspiring product owner or data professional. SQL (Structured Query Language) is the standard language for interacting with relational databases, but each RDBMS has its own history, strengths, weaknesses, and minor variations in SQL syntax (often called SQL dialects).

This section provides a high-level comparison of some of the most popular RDBMS: PostgreSQL, MySQL, Oracle Database, Microsoft SQL Server, and IBM Db2.

SQL Dialect Differences (Examples)

Understanding that SQL syntax can vary is important. Here are a few common examples:

  1. Auto-incrementing Columns:
    • PostgreSQL: SERIAL or GENERATED ALWAYS AS IDENTITY
    • MySQL: AUTO_INCREMENT
    • SQL Server: IDENTITY(1,1)
    • Oracle: GENERATED AS IDENTITY or Sequences
    • Db2: GENERATED ALWAYS AS IDENTITY
  2. Limiting Results:
    • PostgreSQL/MySQL: LIMIT <count> OFFSET <start>
    • SQL Server: OFFSET <start> ROWS FETCH NEXT <count> ROWS ONLY (SQL 2012+) or TOP <count> (older versions)
    • Oracle: FETCH FIRST <count> ROWS ONLY (12c+) or ROWNUM <= <count> (older versions)
    • Db2: FETCH FIRST <count> ROWS ONLY
  1. String Concatenation:
    • PostgreSQL/Oracle/Db2: || operator
    • MySQL: CONCAT() function
    • SQL Server: + operator or CONCAT() function
  2. Date/Time Functions:
    • PostgreSQL: NOW(), CURRENT_DATE, EXTRACT(EPOCH FROM ...)
    • MySQL: NOW(), CURDATE(), UNIX_TIMESTAMP()
    • SQL Server: GETDATE(), CAST(GETDATE() AS DATE), DATEDIFF(s, ...)
    • Oracle: SYSDATE, TRUNC(SYSDATE), (date1 - date2) * 86400
    • Db2: CURRENT TIMESTAMP, CURRENT DATE
  3. Conditional Logic in Queries:
    • Standard SQL (All): CASE WHEN ... THEN ... ELSE ... END
    • PostgreSQL/Oracle/Db2: DECODE() (Oracle/Db2 specific, less common now)
    • MySQL: IF(condition, true_val, false_val), IFNULL(val1, val2)
    • SQL Server: IIF(condition, true_val, false_val), ISNULL(val1, val2)
    • PostgreSQL: COALESCE(val1, val2, ...) (Standard), NULLIF(val1, val2) (Standard)

Why Focus on PostgreSQL Here?

While acknowledging the importance of other RDBMS, this training uses PostgreSQL for hands-on work due to:

  • Open Source Nature: Freely available for learning and development.
  • SQL Compliance: Adheres closely to SQL standards, making learned concepts highly transferable.
  • Rich Feature Set: Supports advanced features like JSONB, window functions, and extensibility, which are relevant in modern applications.
  • Growing Popularity: Increasingly chosen for new projects.
  • Your Specific Environment: Aligns with the planned Neon database setup.

Throughout the course, we will point out significant differences in syntax or behavior compared to other major databases where relevant, ensuring you gain a broad understanding of SQL principles applicable across various platforms.
