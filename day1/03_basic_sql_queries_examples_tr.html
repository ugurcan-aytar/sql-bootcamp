Temel SQL Sorgulari - Pratik Ornekler

Alistirma 1: Temel SELECT Ifadeleri

Alistirma 1.1: Belirli Sutunlari Secme

res_partner tablosundan yalnizca name ve email sutunlarini getiren bir sorgu yazin.

Ipucu: SELECT ifadesinden sonra virgullerle ayrilmis sutun adlarini kullanin. 
SELECT name, email
FROM res_partner;

Alistirma 1.2: Sutun Takma Adlari Kullanma

product_template tablosundan name sutununu "Urun Adi" ve list_price sutununu "Fiyat (USD)" olarak getirin.

Ipucu: Bosluk iceren takma adlar icin AS anahtar kelimesini cift tirnak ile kullanin. 
SELECT
    name AS "Urun Adi",
    list_price AS "Fiyat (USD)"
FROM product_template;

Alistirma 1.3: SELECT'te Ifadeler

Tum urunler icin %10 indirim hesaplayin. Urun adini, orijinal fiyati ve indirimli fiyati goruntuleyin.

Ipucu: SELECT yan tumcesinde dogrudan aritmetik islemler yapabilirsiniz. %10 indirim icin fiyati 0.9 ile carpin. 
SELECT
    name,
    list_price AS orijinal_fiyat,
    list_price * 0.9 AS indirimli_fiyat
FROM product_template;

Alistirma 1.4: DISTINCT Degerler

res_partner tablosundan tum benzersiz ulke kimliklerini (country_id) bulun.

SELECT DISTINCT country_id
FROM res_partner;

Alistirma 2: WHERE ile Filtreleme

Alistirma 2.1: Basit Karsilastirma

Fiyati $100'dan fazla olan tum urunleri bulun.

SELECT name, list_price
FROM product_template
WHERE list_price > 100;

Alistirma 2.2: AND Operatoru Kullanma

Fiyati $50 ile $200 arasinda olan tum aktif urunleri bulun.

SELECT name, list_price, active
FROM product_template
WHERE active = true
  AND list_price >= 50
  AND list_price <= 200;

BETWEEN kullanan alternatif:

SELECT name, list_price, active
FROM product_template
WHERE active = true
  AND list_price BETWEEN 50 AND 200;

Alistirma 2.3: OR Operatoru Kullanma

Amerika Birlesik Devletleri (country_id = 1) VEYA Kanada'dan (country_id = 2) ortaklari bulun.

SELECT name, city, country_id
FROM res_partner
WHERE country_id = 1 OR country_id = 2;

IN kullanan alternatif:

SELECT name, city, country_id
FROM res_partner
WHERE country_id IN (1, 2);

Alistirma 2.4: NULL Degerlerle Calisma

E-posta adresi OLMAYAN tum ortaklari bulun.

Ipucu: Unutmayin, NULL ile karsilastirmak icin = kullanamazsiniz. Bunun yerine IS NULL kullanin. 
SELECT name, email
FROM res_partner
WHERE email IS NULL;

Alistirma 2.5: NOT Operatoru

Iptal edilmemis tum satis siparislerini bulun.

SELECT name, state, amount_total
FROM sale_order
WHERE NOT state = 'cancel';

Alternatif:

SELECT name, state, amount_total
FROM sale_order
WHERE state <> 'cancel';

Alistirma 3: LIKE ile Desen Eslestirme

Alistirma 3.1: Ile Baslayan Deseni

Adi "G" ile baslayan tum ortaklari bulun.

SELECT name, email
FROM res_partner
WHERE name LIKE 'G%';

Alistirma 3.2: Ile Biten Deseni

E-postasi ".org" ile biten tum ortaklari bulun.

SELECT name, email
FROM res_partner
WHERE email LIKE '%.org';

Alistirma 3.3: Iceren Deseni

Adinda "Wireless" kelimesi gecen tum urunleri bulun.

SELECT name, list_price
FROM product_template
WHERE name LIKE '%Wireless%';

Alistirma 3.4: Tek Karakter Joker Karakteri

Ikinci karakteri "M" olan tum urun kodlarini bulun (ornegin, "WM-002", "KM-005").

Ipucu: Tam olarak bir karakteri eslemek icin alt cizgi (_) kullanin. 
SELECT name, default_code
FROM product_template
WHERE default_code LIKE '_M%';

Alistirma 4: ORDER BY ile Siralama

Alistirma 4.1: Artan Siralama

Tum ortaklari ada gore alfabetik olarak listeleyin.

SELECT name, city
FROM res_partner
ORDER BY name ASC;

Not: ASC varsayilan siralama duzenoldugu icin istege baglidir.

Alistirma 4.2: Azalan Siralama

Tum urunleri en yuksekten en dusuge dogru fiyata gore listeleyin.

SELECT name, list_price
FROM product_template
ORDER BY list_price DESC;

Alistirma 4.3: Birden Fazla Sutuna Gore Siralama

Tum satis siparislerini duruma gore (alfabetik olarak) ve her durum icinde tutara gore (en yuksekten ilk) listeleyin.

SELECT name, state, amount_total
FROM sale_order
ORDER BY state ASC, amount_total DESC;

Alistirma 5: Sonuclari Sinirlandirma

Alistirma 5.1: Ilk N Sonuc

En pahali 3 urunu bulun.

-- PostgreSQL/MySQL sozdizimi
SELECT name, list_price
FROM product_template
ORDER BY list_price DESC
LIMIT 3;

Standart SQL sozdizimi:

SELECT name, list_price
FROM product_template
ORDER BY list_price DESC
FETCH FIRST 3 ROWS ONLY;

Alistirma 5.2: OFFSET ile Sayfalama

Alfabetik olarak sirali ortaklarin ikinci sayfasini (sayfa basina 3) alin.

Ipucu: Sayfa basina 3 oge ile 2. sayfa icin ilk 3 kaydi atlamaniz gerekir. 
-- PostgreSQL/MySQL sozdizimi
SELECT name, city
FROM res_partner
ORDER BY name
LIMIT 3 OFFSET 3;

Standart SQL sozdizimi:

SELECT name, city
FROM res_partner
ORDER BY name
OFFSET 3 ROWS FETCH NEXT 3 ROWS ONLY;

Meydan Okuma Alistirmalari

Bu alistirmalar dersten birden fazla kavrami bir araya getirmektedir. Cozumleri kontrol etmeden once bunlari cozmeye calisin.

Meydan Okuma 1: Yuksek Degerli Aktif Siparisler

Tutari $500'dan fazla olan tum onaylanmis (state = 'sale') veya tamamlanmis (state = 'done') siparisleri bulun. Siparis adini, tutarini ve durumunu goruntuleyin. Tutara gore azalan sirada siralayin ve ilk 5 ile sinirlandin.

SELECT
    name AS siparis_ref,
    amount_total,
    state
FROM sale_order
WHERE
    state IN ('sale', 'done')
    AND amount_total > 500
ORDER BY amount_total DESC
LIMIT 5;

Meydan Okuma 2: Ortak Iletisim Ozeti

Ortak adlarini iletisim bilgileriyle gosteren bir rapor olusturun. Adi ("Iletisim Adi" olarak), e-postayi ("E-posta Adresi" olarak) ve telefonu ekleyin. Yalnizca HEM e-postasi HEM DE telefonu olan (NULL olmayan) ortaklari gosterin. Ada gore alfabetik olarak siralayin.

SELECT
    name AS "Iletisim Adi",
    email AS "E-posta Adresi",
    phone
FROM res_partner
WHERE
    email IS NOT NULL
    AND phone IS NOT NULL
ORDER BY name;

Meydan Okuma 3: Urun Fiyat Analizi

Tum aktif urunleri su sekilde listeleyin: ad, orijinal fiyat, %15 indirimli fiyat ("Indirimli Fiyat" olarak) ve indirim tutari ("Kazanciniz" olarak). Yalnizca $50 ve uzerindeki urunleri dahil edin. Indirim tutarina gore (en yuksek tasarruf ilk), 5 ile sinirlandin.

SELECT
    name,
    list_price AS orijinal_fiyat,
    list_price * 0.85 AS "Indirimli Fiyat",
    list_price * 0.15 AS "Kazanciniz"
FROM product_template
WHERE
    active = true
    AND list_price >= 50
ORDER BY "Kazanciniz" DESC
LIMIT 5;

Meydan Okuma 4: E-posta Alan Adi Aramasi

E-postasi "example.com" veya "sample.org" alan adi olan tum ortaklari bulun. Adlarini, e-postalarini ve sehirlerini gosterin. E-postasi olmayan ortaklari haric turun. E-posta alan adina gore siralayin (boylece once tum example.com, sonra sample.org gorunur), sonra ada gore.

SELECT
    name,
    email,
    city
FROM res_partner
WHERE
    email IS NOT NULL
    AND (
        email LIKE '%@example.com'
        OR email LIKE '%@sample.org'
    )
ORDER BY
    email ASC,
    name ASC;

Meydan Okuma 5: Urun Kodu Deseni

Urun kodunun (default_code) iki harf, tire ve uc rakam desenini takip ettigi tum urunleri bulun (ornegin, "LP-001"). Adi ve kodu gosterin, koda gore siralayin.

Ipucu: Birden fazla tek karakter icin birden fazla alt cizgi kullanin. Desen soyle olacaktir: iki karakter, tire, uc karakter. 
SELECT name, default_code
FROM product_template
WHERE default_code LIKE '__-___'
ORDER BY default_code;

Not: Bu tam olarak 2 karakter, tire ve 3 karakteri esler. Tum ornek urunler bu desene uymaktadir.

Onemli Cikarimlar

  • SELECT: SELECT * kullanmak yerine her zaman yalnizca ihtiyaciniz olan sutunlari belirtin
  • Takma Adlar: Sonuclarda okunabilir sutun adlari saglamak icin AS kullanin
  • Ifadeler: SELECT yan tumcesinde dogrudan hesaplamalar yapabilirsiniz
  • DISTINCT: Sonuclarinizdan yinelenen satirlari kaldirir
  • WHERE: Herhangi bir gruplamadan once kosullara gore satirlari filtreler
  • NULL kullanimi: Her zaman IS NULL veya IS NOT NULL kullanin (asla = NULL degil)
  • LIKE: Herhangi bir karakter icin % ve tam olarak bir karakter icin _ kullanin
  • ORDER BY: Sonuclari siralar; azalan siralama icin DESC kullanin
  • LIMIT/FETCH: Dondrulen satir sayisini kisitlar; tahmin edilebilir sonuclar icin her zaman ORDER BY ile kullanin
