Exercise 1: Basic SELECT Statements

Exercise 1.1: Selecting Specific Columns

Write a query to retrieve only the name and email columns from the res_partner table.

Hint: Use SELECT followed by the column names separated by commas. 
SELECT name, email
FROM res_partner;

Expected Result: 8 rows showing partner names and their email addresses (including NULL values).

Exercise 1.2: Using Column Aliases

Retrieve the name as "Product Name" and list_price as "Price (USD)" from product_template.

Hint: Use the AS keyword with double quotes for aliases containing spaces. 
SELECT
    name AS "Product Name",
    list_price AS "Price (USD)"
FROM product_template;

Exercise 1.3: Expressions in SELECT

Calculate a 10% discount on all products. Display the product name, original price, and discounted price.

Hint: You can perform arithmetic operations directly in the SELECT clause. Multiply the price by 0.9 for 10% off. 
SELECT
    name,
    list_price AS original_price,
    list_price * 0.9 AS discounted_price
FROM product_template;

Exercise 1.4: DISTINCT Values

Find all unique country IDs from the res_partner table.

SELECT DISTINCT country_id
FROM res_partner;

Expected Result: 7 unique country IDs (233, 73, 195, 68, 20, 107, 229).

Exercise 2: Filtering with WHERE

Exercise 2.1: Simple Comparison

Find all products with a price greater than $100.

SELECT name, list_price
FROM product_template
WHERE list_price > 100;

Expected Result: Laptop Pro 15, Monitor 27 inch, Keyboard Mechanical, Headphones Wireless.

Exercise 2.2: Using AND Operator

Find all active products with a price between $50 and $200.

SELECT name, list_price, active
FROM product_template
WHERE active = true
  AND list_price >= 50
  AND list_price <= 200;

Alternative using BETWEEN:

SELECT name, list_price, active
FROM product_template
WHERE active = true
  AND list_price BETWEEN 50 AND 200;

Exercise 2.3: Using OR Operator

Find partners from the United States (country_id = 233) OR Canada (country_id = 20).

SELECT name, city, country_id
FROM res_partner
WHERE country_id = 233 OR country_id = 20;

Alternative using IN:

SELECT name, city, country_id
FROM res_partner
WHERE country_id IN (233, 20);

Expected Result: George Miller, John Smith, Sarah Johnson.

Exercise 2.4: Handling NULL Values

Find all partners who do NOT have an email address.

Hint: Remember, you cannot use = to compare with NULL. Use IS NULL instead. 
SELECT name, email
FROM res_partner
WHERE email IS NULL;

Expected Result: John Smith.

Exercise 2.5: NOT Operator

Find all sales orders that are NOT cancelled.

SELECT name, state, amount_total
FROM sale_order
WHERE NOT state = 'cancel';

Alternative:

SELECT name, state, amount_total
FROM sale_order
WHERE state <> 'cancel';

Exercise 3: Pattern Matching with LIKE

Exercise 3.1: Starts With Pattern

Find all partners whose names start with "G".

SELECT name, email
FROM res_partner
WHERE name LIKE 'G%';

Expected Result: George Miller, Gerard Dupont.

Exercise 3.2: Ends With Pattern

Find all partners with emails ending in ".org".

SELECT name, email
FROM res_partner
WHERE email LIKE '%.org';

Expected Result: Anna Schmidt, Sarah Johnson.

Exercise 3.3: Contains Pattern

Find all products whose names contain the word "Wireless".

SELECT name, list_price
FROM product_template
WHERE name LIKE '%Wireless%';

Expected Result: Wireless Mouse, Headphones Wireless.

Exercise 3.4: Single Character Wildcard

Find all product codes that have "M" as the second character (e.g., "WM-002", "KM-005").

Hint: Use the underscore (_) to match exactly one character. 
SELECT name, default_code
FROM product_template
WHERE default_code LIKE '_M%';

Expected Result: Wireless Mouse (WM-002), Keyboard Mechanical (KM-005).

Exercise 4: Sorting with ORDER BY

Exercise 4.1: Ascending Sort

List all partners alphabetically by name.

SELECT name, city
FROM res_partner
ORDER BY name ASC;

Note: ASC is optional as it's the default sort order.

Exercise 4.2: Descending Sort

List all products sorted by price from highest to lowest.

SELECT name, list_price
FROM product_template
ORDER BY list_price DESC;

Exercise 4.3: Multiple Column Sort

List all sales orders sorted by state (alphabetically), and within each state, by amount (highest first).

SELECT name, state, amount_total
FROM sale_order
ORDER BY state ASC, amount_total DESC;

Exercise 5: Limiting Results

Exercise 5.1: Top N Results

Find the 3 most expensive products.

-- PostgreSQL/MySQL syntax
SELECT name, list_price
FROM product_template
ORDER BY list_price DESC
LIMIT 3;

Standard SQL syntax:

SELECT name, list_price
FROM product_template
ORDER BY list_price DESC
FETCH FIRST 3 ROWS ONLY;

Expected Result: Laptop Pro 15, Monitor 27 inch, Headphones Wireless.

Exercise 5.2: Pagination with OFFSET

Get the second page of partners (3 per page), sorted alphabetically.

Hint: For page 2 with 3 items per page, you need to skip the first 3 records. 
-- PostgreSQL/MySQL syntax
SELECT name, city
FROM res_partner
ORDER BY name
LIMIT 3 OFFSET 3;

Standard SQL syntax:

SELECT name, city
FROM res_partner
ORDER BY name
OFFSET 3 ROWS FETCH NEXT 3 ROWS ONLY;

Challenge Exercises

These exercises combine multiple concepts from the lesson. Try to solve them before checking the solutions.

Challenge 1: High-Value Active Orders

Find all confirmed (state = 'sale') or completed (state = 'done') orders with an amount greater than $500. Display the order name, amount, and state. Sort by amount descending and limit to top 5.

SELECT
    name AS order_ref,
    amount_total,
    state
FROM sale_order
WHERE
    state IN ('sale', 'done')
    AND amount_total > 500
ORDER BY amount_total DESC
LIMIT 5;

Challenge 2: Partner Contact Summary

Create a report showing partner names with their contact info. Include name (as "Contact Name"), email (as "Email Address"), and phone. Only show partners who have BOTH email AND phone (not NULL). Sort alphabetically by name.

SELECT
    name AS "Contact Name",
    email AS "Email Address",
    phone
FROM res_partner
WHERE
    email IS NOT NULL
    AND phone IS NOT NULL
ORDER BY name;

Challenge 3: Product Price Analysis

List all active products showing: name, original price, 15% discounted price (as "Sale Price"), and the discount amount (as "You Save"). Only include products priced $50 or more. Sort by discount amount (highest savings first), limit to 5.

SELECT
    name,
    list_price AS original_price,
    list_price * 0.85 AS "Sale Price",
    list_price * 0.15 AS "You Save"
FROM product_template
WHERE
    active = true
    AND list_price >= 50
ORDER BY "You Save" DESC
LIMIT 5;

Challenge 4: Email Domain Search

Find all partners whose email is from either "example.com" or "sample.org" domain. Show their name, email, and city. Exclude partners without email. Sort by email domain (so all example.com appear first, then sample.org), then by name.

SELECT
    name,
    email,
    city
FROM res_partner
WHERE
    email IS NOT NULL
    AND (
        email LIKE '%@example.com'
        OR email LIKE '%@sample.org'
    )
ORDER BY
    email ASC,
    name ASC;

Challenge 5: Product Code Pattern

Find all products where the product code (default_code) follows the pattern of two letters, a hyphen, and three digits (e.g., "LP-001"). Show the name and code, sorted by code.

Hint: Use multiple underscores for multiple single characters. The pattern would be: two characters, hyphen, three characters. 
SELECT name, default_code
FROM product_template
WHERE default_code LIKE '__-___'
ORDER BY default_code;

Note: This matches exactly 2 characters, hyphen, and 3 characters. All sample products match this pattern.

Key Takeaways

  • SELECT: Always specify only the columns you need rather than using SELECT *
  • Aliases: Use AS to provide readable column names in results
  • Expressions: You can perform calculations directly in the SELECT clause
  • DISTINCT: Removes duplicate rows from your results
  • WHERE: Filters rows based on conditions before any grouping
  • NULL handling: Always use IS NULL or IS NOT NULL (never = NULL)
  • LIKE: Use % for any characters and _ for exactly one character
  • ORDER BY: Sort results; use DESC for descending order
  • LIMIT/FETCH: Restrict the number of returned rows; always use with ORDER BY for predictable results
