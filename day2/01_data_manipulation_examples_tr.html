Veri Manipulasyonu (DML) - Pratik Ornekler

Bu sayfa, INSERT, UPDATE, DELETE ve islem (transaction) yonetimini uygulamak icin pratik alistirmalar sunmaktadir. Veritabani verilerini guvenli bir sekilde degistirme konusunda guven kazanmak icin bu alistirmalar uzerinde calisin.

Onemli Guvenlik Notu: DML islemlerini her zaman test/gelistirme veritabaninda uygulayiniz. Uretim ortaminda her zaman islemleri (transaction) kullanin ve WHERE kosullarinizi once SELECT ile test edin!

Alistirma 1: INSERT Ifadeleri

Alistirma 1.1: Tek Satir Ekleme

Yeni bir urun ekleyin: Kategori 1'de "Mechanical Keyboard", fiyati $79.99, stokta 75 adet ve aktif.

INSERT INTO products (product_name, category_id, price, stock_quantity, active)
VALUES ('Mechanical Keyboard', 1, 79.99, 75, true);

Alistirma 1.2: Birden Fazla Satir Ekleme

Tek bir ifadede uc yeni musteri ekleyin:

  • Frank Miller, frank@email.com, aktif
  • Grace Lee, grace@email.com, aktif
  • Henry Chen, henry@email.com, aktif

INSERT INTO customers (name, email, status, created_at)
VALUES
    ('Frank Miller', 'frank@email.com', 'active', CURRENT_DATE),
    ('Grace Lee', 'grace@email.com', 'active', CURRENT_DATE),
    ('Henry Chen', 'henry@email.com', 'active', CURRENT_DATE);

Alistirma 1.3: DEFAULT ile Ekleme

Ust kategorisi olmayan (NULL) yeni bir "Accessories" kategorisi ekleyin. category_id'nin otomatik olusturuldugunu varsayin.

INSERT INTO categories (category_name, parent_id)
VALUES ('Accessories', NULL);

Acik DEFAULT ile alternatif:

INSERT INTO categories (category_name, parent_id)
VALUES ('Accessories', DEFAULT);

Alistirma 1.4: SELECT'ten Ekleme

Tum Elektronik urunlerinin (category_id = 1) yedegini, ayni yapiya sahip bir products_backup tablosuna ekleyerek olusturun.

Ipucu: Bir tablodan digerine veri kopyalamak icin INSERT INTO ... SELECT kullanin. 
INSERT INTO products_backup (product_name, category_id, price, stock_quantity, active)
SELECT product_name, category_id, price, stock_quantity, active
FROM products
WHERE category_id = 1;

Alistirma 1.5: RETURNING ile Ekleme (PostgreSQL)

Yeni bir urun ekleyin ve olusturulan product_id'yi urun adiyla birlikte donduren bir sorgu yazin.

-- PostgreSQL sozdizimi
INSERT INTO products (product_name, category_id, price, stock_quantity, active)
VALUES ('Webcam HD', 1, 89.99, 40, true)
RETURNING product_id, product_name;
MySQL alternatifi: INSERT'ten sonra SELECT LAST_INSERT_ID(); kullanin.

Alistirma 2: UPDATE Ifadeleri

Kritik Guvenlik Kurali

UPDATE calistirmadan once WHERE kosulunuzu HER ZAMAN bir SELECT ifadesiyle test edin!

-- Adim 1: Hangi satirlarin etkilenecegini test edin
SELECT * FROM products WHERE category_id = 1;

-- Adim 2: Dogru ise UPDATE'i calistirin
UPDATE products SET price = price * 1.10 WHERE category_id = 1;

Alistirma 2.1: Basit UPDATE

Eve Wilson'in e-postasini 'eve@old-email.com'dan 'eve@new-email.com'a guncelleyin.

-- Once hangi satirin etkilenecegini dogrulayin
SELECT * FROM customers WHERE customer_id = 5;

-- Sonra guncelleyin
UPDATE customers
SET email = 'eve@new-email.com'
WHERE customer_id = 5;

Alistirma 2.2: Hesaplama ile UPDATE

Tum Elektronik urunlerinin (category_id = 1) fiyatini %15 artirin.

UPDATE products
SET price = price * 1.15
WHERE category_id = 1;

Sonra

Alistirma 2.3: Birden Fazla Sutunu Guncelleme

"Desk Lamp" urununu yeniden aktif edin: active'i true ve stock_quantity'yi 25 olarak ayarlayin.

UPDATE products
SET active = true,
    stock_quantity = 25
WHERE product_id = 5;

Alistirma 2.4: Mevcut Degere Gore UPDATE

product_id = 2 icin stock_quantity'yi 1 azaltin, ancak sadece stock_quantity 0'dan buyukse.

UPDATE products
SET stock_quantity = stock_quantity - 1
WHERE product_id = 2
  AND stock_quantity > 0;

Not: Ekstra kosul, negatif stok degerlerini onler.

Alistirma 2.5: Baska Tabloya Dayali UPDATE (PostgreSQL)

Pasif musterilerin tum siparislerini 'cancelled' olarak isaretleyin.

Ipucu: customers tablosuyla birlestirmek icin FROM yan tumcesini (PostgreSQL) kullanin. 
-- PostgreSQL sozdizimi
UPDATE orders o
SET status = 'cancelled'
FROM customers c
WHERE o.customer_id = c.customer_id
  AND c.status = 'inactive';

Alt sorgu kullanan alternatif (cogu veritabaninda calisir):

UPDATE orders
SET status = 'cancelled'
WHERE customer_id IN (
    SELECT customer_id FROM customers WHERE status = 'inactive'
);

Alistirma 3: DELETE Ifadeleri

Kritik Guvenlik Kurali

WHERE kosulunuzu HER ZAMAN once SELECT ile test edin! WHERE olmadan bir DELETE TUM satirlari silecektir!

Alistirma 3.1: Tek Satir Silme

Iptal edilen siparisi (order_id = 104) silin.

-- Once dogrulayin
SELECT * FROM orders WHERE order_id = 104;

-- Sonra silin
DELETE FROM orders
WHERE order_id = 104;

Alistirma 3.2: Kosullu Silme

2022'den once olusturulan tum pasif musterileri silin.

-- Once hangi satirlarin silinecegini dogrulayin
SELECT * FROM customers
WHERE status = 'inactive'
  AND created_at < '2022-01-01';

-- Sonra silin
DELETE FROM customers
WHERE status = 'inactive'
  AND created_at < '2022-01-01';

Alistirma 3.3: Baska Tabloya Dayali Silme

'inactive' olarak isaretlenen musterilerin tum siparislerini silin.

-- Alt sorgu kullanarak (cogu veritabaninda calisir)
DELETE FROM orders
WHERE customer_id IN (
    SELECT customer_id FROM customers WHERE status = 'inactive'
);

USING ile PostgreSQL alternatifi:

DELETE FROM orders o
USING customers c
WHERE o.customer_id = c.customer_id
  AND c.status = 'inactive';

Alistirma 3.4: TRUNCATE ve DELETE Karsilastirmasi

Bu iki ifade arasindaki fark nedir? Her birini ne zaman kullanirsiniz?

-- Secenek A
DELETE FROM temp_data;

-- Secenek B
TRUNCATE TABLE temp_data;

DELETE kullanin: Kosullu silme gerektiginde, tetikleyiciler calistirilmasi gerektiginde veya geri alma gerektiginde.

TRUNCATE kullanin: Bir tabloyu (gecici/hazirlik tablolari gibi) hizlica temizlemek istediginizde ve yukaridaki ozelliklere ihtiyaciniz olmadiginda.

Alistirma 4: Islemler (Transactions)

Islem Temelleri

  • BEGIN: Bir islem baslatir
  • COMMIT: Tum degisiklikleri kalici olarak kaydeder
  • ROLLBACK: BEGIN'den bu yana yapilan tum degisiklikleri geri alir
  • SAVEPOINT: Islem icinde bir kontrol noktasi olusturur

Alistirma 4.1: Temel Islem

A001 hesabindan A002 hesabina $500 aktaran bir islem yazin. Her iki islem de basarili olmali veya her ikisi de basarisiz olmalidir.

BEGIN;

-- Kaynak hesaptan dus
UPDATE accounts
SET balance = balance - 500.00
WHERE account_id = 'A001';

-- Hedef hesaba ekle
UPDATE accounts
SET balance = balance + 500.00
WHERE account_id = 'A002';

-- Her sey dogruysa, kaydet
COMMIT;

-- Bir seyler yanlis gittiyse, bunun yerine ROLLBACK kullanin
-- ROLLBACK;
Once

A001: $5000.00
A002: $3500.00

COMMIT Sonrasi

A001: $4500.00
A002: $4000.00

Alistirma 4.2: Dogrulama ile Islem

Sadece kaynak hesapta yeterli bakiye varsa para aktaran bir islem yazin. Yoksa, geri alin.

Ipucu: Kesintiden once veya sonra bakiyeyi kontrol edin ve COMMIT veya ROLLBACK'e karar verin. 
BEGIN;

-- Dusmeyi dene (yeterli bakiye kontrolu ile)
UPDATE accounts
SET balance = balance - 500.00
WHERE account_id = 'A001'
  AND balance >= 500.00;  -- Sadece yeterli bakiye varsa guncelle

-- Guncellemenin gercekten yapilip yapilmadigini kontrol edin (uygulama kodunda)
-- 0 satir etkilendiyse, ROLLBACK
-- 1 satir etkilendiyse, devam edin:

UPDATE accounts
SET balance = balance + 500.00
WHERE account_id = 'A002';

COMMIT;

Not: Gercek uygulamalarda, programlama dilinizde etkilenen satir sayisini kontrol eder ve kosullu olarak COMMIT veya ROLLBACK yaparsaniz.

Alistirma 4.3: SAVEPOINT Kullanimi

Iki urun ekleyen bir islem olusturun. Ilk eklemeden sonra bir SAVEPOINT kullanin. Ikinci ekleme basarisiz olursa, sadece savepoint'e geri alin (ilk eklemeyi koruyarak).

BEGIN;

-- Ilk ekleme
INSERT INTO products (product_name, category_id, price, stock_quantity, active)
VALUES ('Product A', 1, 19.99, 100, true);

-- Savepoint olustur
SAVEPOINT after_first_insert;

-- Ikinci ekleme (basarisiz olabilir)
INSERT INTO products (product_name, category_id, price, stock_quantity, active)
VALUES ('Product B', 1, 29.99, 50, true);

-- Ikinci ekleme basarisiz olursa:
-- ROLLBACK TO SAVEPOINT after_first_insert;

-- Kaydet (savepoint'e geri aldysak bile ilk urun kaydedilecek)
COMMIT;

Alistirma 5: UPSERT Islemleri

Alistirma 5.1: INSERT veya UPDATE (PostgreSQL)

product_id = 100 ile bir urun ekleyin. Zaten varsa, bunun yerine fiyatini ve stock_quantity'sini guncelleyin.

-- PostgreSQL sozdizimi
INSERT INTO products (product_id, product_name, category_id, price, stock_quantity, active)
VALUES (100, 'New Product', 1, 49.99, 75, true)
ON CONFLICT (product_id)
DO UPDATE SET
    price = EXCLUDED.price,
    stock_quantity = EXCLUDED.stock_quantity;
EXCLUDED eklenmeye calisilan degerleri ifade eder.

Alistirma 5.2: INSERT veya IGNORE

Bir urun eklemeyi deneyin. Catisma varsa (urun zaten mevcutsa), hicbir sey yapmayin.

-- PostgreSQL sozdizimi
INSERT INTO products (product_id, product_name, category_id, price, stock_quantity, active)
VALUES (100, 'New Product', 1, 49.99, 75, true)
ON CONFLICT (product_id)
DO NOTHING;

Alistirma 5.3: Artirma ile UPSERT

Bir urun envanter kaydi ekleyin. Zaten varsa, yeni miktari mevcut miktara ekleyin.

-- PostgreSQL sozdizimi
INSERT INTO products (product_id, product_name, category_id, price, stock_quantity, active)
VALUES (2, 'Wireless Mouse', 1, 29.99, 50, true)
ON CONFLICT (product_id)
DO UPDATE SET
    stock_quantity = products.stock_quantity + EXCLUDED.stock_quantity;

Sonuc: Urun 2'nin stokta 200 adeti varsa, simdi 250 olur.

Meydan Okuma Alistirmalari

Meydan Okuma 1: Toplu Fiyat Guncelleme

Mobilya kategorisindeki (category_id = 2) stokta bulunan (stock_quantity > 0) tum urunlere %20 indirim uygulayan bir ifade yazin. Guvenlik icin bir islem kullanin.

BEGIN;

-- Once neyin etkilenecegini gorun
SELECT product_name, price, price * 0.80 AS new_price
FROM products
WHERE category_id = 2
  AND stock_quantity > 0;

-- Indirimi uygula
UPDATE products
SET price = price * 0.80
WHERE category_id = 2
  AND stock_quantity > 0;

COMMIT;

Meydan Okuma 2: Arsivle ve Sil

Sunlari yapan ifadeler yazin: (1) Tum iptal edilmis siparisleri bir orders_archive tablosuna kopyalayin, sonra (2) Iptal edilmis siparisleri ana orders tablosundan silin. Bir islem kullanin.

BEGIN;

-- Adim 1: Iptal edilmis siparisleri arsive kopyala
INSERT INTO orders_archive (order_id, customer_id, order_date, total_amount, status)
SELECT order_id, customer_id, order_date, total_amount, status
FROM orders
WHERE status = 'cancelled';

-- Adim 2: Ana tablodan sil
DELETE FROM orders
WHERE status = 'cancelled';

COMMIT;

Meydan Okuma 3: Envanter Ayarlamasi

Bir siparisi isleyen bir islem yazin: product_id = 1 icin stock_quantity'yi 5 birim azaltin, ancak sadece yeterli stok varsa. Basarili olursa, siparis durumunu da 'shipped' olarak guncelleyin. Yetersiz stok varsa, her seyi geri alin.

BEGIN;

-- Stoku dusurmeyi dene (sadece yeterliyse)
UPDATE products
SET stock_quantity = stock_quantity - 5
WHERE product_id = 1
  AND stock_quantity >= 5;

-- Uygulama kodunda: 1 satir etkilenip etkilenmedigini kontrol edin
-- Evetse, devam edin:

UPDATE orders
SET status = 'shipped'
WHERE order_id = 103;

COMMIT;

-- Ilk UPDATE'te satir etkilenmediyse, kullanin:
-- ROLLBACK;

Meydan Okuma 4: Toplu Musteri Temizligi

orders tablosunda hic siparis vermemis tum musterileri 'inactive' olarak ayarlayan bir ifade yazin. Bir alt sorgu veya NOT EXISTS kullanin.

-- NOT EXISTS kullanarak
UPDATE customers c
SET status = 'inactive'
WHERE NOT EXISTS (
    SELECT 1 FROM orders o
    WHERE o.customer_id = c.customer_id
);

-- NOT IN kullanan alternatif
UPDATE customers
SET status = 'inactive'
WHERE customer_id NOT IN (
    SELECT DISTINCT customer_id FROM orders
);

Temel Cikarimlar

  • INSERT: Tek veya birden fazla satir ekleyin; veri kopyalamak icin INSERT...SELECT kullanin
  • UPDATE: Kasitli olarak tum satirlari guncellemiyorsaniz her zaman WHERE kullanin
  • DELETE: Her zaman once SELECT ile test edin; tum verilerin silinmesini onlemek icin WHERE kullanin
  • TRUNCATE: Tum satirlari kaldirmanin hizli yolu; gecici tablolari temizlemek icin kullanin
  • Islemler (Transactions): Iliskili islemleri gruplayin; kaydetmek icin COMMIT, geri almak icin ROLLBACK kullanin
  • SAVEPOINT: Kismi geri alma icin islemler icinde kontrol noktalari olusturun
  • UPSERT: Catismaya gore INSERT veya UPDATE; sozdizimi veritabanina gore degisir
  • Oncelik Guvenlik: Her zaman gelistirme ortaminda test edin, islemleri kullanin, UPDATE/DELETE oncesi SELECT ile dogrulayin
En Iyi Uygulama Is Akisi:
  1. WHERE kosulunuzu yazin
  2. Etkilenen satirlari gormek icin SELECT ile test edin
  3. BEGIN islemi icine sarin
  4. UPDATE/DELETE'i calistirin
  5. Sonuclari dogrulayin
  6. Dogruysa COMMIT, degilse ROLLBACK yapin
