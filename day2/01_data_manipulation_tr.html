Gun 2: Veri Manipulasyonu (DML)

Veri Manipulasyonu

Bu oturum, tablolara veri eklemek, degistirmek ve silmek icin kullanilan SQL komutlarini kapsar: INSERT, UPDATE ve DELETE. Ayrica bu islemler sirasinda veri butunlugunu saglamak icin kritik oneme sahip islem (transaction) yonetimini de tartisacagiz.

INSERT Ifadeleri (Veri Ekleme)

INSERT ifadesi bir tabloya yeni satirlar (kayitlar) ekler.

Standart SQL Sozdizimi:

  1. Tek Satir Ekleme: Tabloyu, doldurulacak sutunlari ve karsilik gelen degerleri belirtin. 

    -- Tek satir ekle
INSERT INTO table_name (column1, column2) VALUES (value1, value2);

    • Eger tum sutunlar icin tanimli siralarinda deger sagliyorsaniz, sutun listesi bazen atlanabilir, ancak sutunlari acikca listelemek daha guvenli ve okunabilirdir.

  2. Birden Fazla Satir Ekleme: Cogu veritabani, birden fazla deger demeti kullanarak tek bir INSERT ifadesiyle birden fazla satir eklemeyi destekler. 

    -- Birden fazla satir ekle
INSERT INTO table_name (column1, column2) VALUES
(value1a, value2a),
(value1b, value2b),
(value1c, value2c);

  3. Sorgudan Ekleme: Bir SELECT ifadesinin sonucuna dayali olarak satirlar ekleyin. 

    -- Birden fazla satir ekle
INSERT INTO table_name (column1, column2)
SELECT source_column1, source_column2
FROM source_table
WHERE condition;

Varsayilan Degerlerin Islenmesi (Standart SQL):

Eger bir sutunun tanimli bir varsayilan degeri varsa, onu INSERT ifadesinden cikarabilir (sutunlari acikca listelerseniz) veya DEFAULT anahtar kelimesini kullanabilirsiniz.

-- column3'un varsayilan bir degeri oldugunu varsayarak
INSERT INTO table_name (column1, column2) VALUES (value1, value2); -- column3'u atlayarak

INSERT INTO table_name (column1, column2, column3) 
VALUES (value1, value2, DEFAULT); -- Acikca DEFAULT kullanarak

Olusturulan Degerleri Alma (Sozdizimi Degisir):

Cogu zaman, bir eklemeden sonra otomatik olusturulan bir anahtarin (ID gibi) degerini almaniz gerekir.

  • PostgreSQL/Oracle: RETURNING yan tumcesini kullanin. 
    -- PostgreSQL/Oracle Ornegi
INSERT INTO products (product_name, price)
VALUES ("New Gadget", 99.99)
RETURNING product_id;
  • SQL Server: OUTPUT yan tumcesini kullanin. 
    -- SQL Server Ornegi
INSERT INTO products (product_name, price)
OUTPUT inserted.product_id
VALUES ("New Gadget", 99.99);
  • MySQL: Sonraki bir sorguda (ayni oturum icinde) LAST_INSERT_ID() fonksiyonunu kullanin. 
    -- MySQL Ornegi
INSERT INTO products (product_name, price)
VALUES ("New Gadget", 99.99);
SELECT LAST_INSERT_ID();
  • Standart SQL: IDENTITY() fonksiyonu tanimlidir, ancak satici destegi ve kullanimi degisir.

Ornekler (Odoo/PostgreSQL Semasini Kullanarak):

-- Tek bir urun kategorisi ekle
INSERT INTO product_category (name, parent_id) VALUES ("Wearables", 1);

-- Birden fazla partner ekle
INSERT INTO res_partner (name, email, customer_rank)
VALUES 
    ("Alice Smith", "alice@example.com", 1),
    ("Bob Johnson", "bob@sample.org", 1);

-- DEFAULT ve RETURNING kullanarak ekle (PostgreSQL)
INSERT INTO product_template (name, list_price, type, categ_id, uom_id, uom_po_id, active)
VALUES ("Basic Widget", 19.95, "product", 1, 1, 1, DEFAULT)
RETURNING id, name;

-- Sorgudan ekle (bazi kategorilerin kopyalarini olustur)
INSERT INTO product_category (name, parent_id)
SELECT name || " (Backup)", parent_id
FROM product_category
WHERE id < 5;

UPDATE Ifadeleri (Veri Degistirme)

UPDATE ifadesi bir tablodaki mevcut satirlari degistirir.

Standart SQL Sozdizimi:

UPDATE table_name
SET column1 = value1, 
    column2 = value2, ...
WHERE condition; -- KRITIK: Hangi satirlarin guncelleneceqini filtreler

Onemli: Tablodaki her satiri guncellemeyi kastetmiyorsaniz her zaman bir WHERE yan tumcesi kullanin. WHERE yan tumcesini unutmak yaygin ve tehlikeli bir hatadir.

Ornekler:

-- Standart SQL: Belirli bir musterinin e-postasini guncelle
UPDATE customers
SET email = "new.email@example.com"
WHERE customer_id = 123;

-- Standart SQL: Bir urun kategorisi icin fiyati artir
UPDATE products
SET price = price * 1.10 -- %10 artir
WHERE category_id = 5;

-- Standart SQL: Mevcut degerlere dayali guncelleme
UPDATE inventory
SET quantity = quantity - 1
WHERE product_id = 456 AND quantity > 0;

Baska Bir Tablodan Guncelleme (Sozdizimi Degisir):

Bir tablodaki satirlari baska bir tablodaki degerlere dayali olarak guncellemek yaygindir, ancak sozdizimi onemli olcude degisir.

  • PostgreSQL/SQL Server: UPDATE ifadesinde bir FROM yan tumcesi kullanin. 
    -- PostgreSQL/SQL Server Ornegi: Tedarikci maliyetlerine dayali urun fiyatlarini guncelle
UPDATE products p
SET price = s.cost * 1.5
FROM suppliers s
WHERE p.supplier_id = s.supplier_id AND p.category_id = 10;
  • Oracle/Db2: Iliskili alt sorgu veya MERGE kullanin. 
    -- Oracle Ornegi (Iliskili Alt Sorgu)
UPDATE products p
SET price = (SELECT s.cost * 1.5 FROM suppliers s WHERE p.supplier_id = s.supplier_id)
WHERE p.category_id = 10 AND EXISTS (SELECT 1 FROM suppliers s WHERE p.supplier_id = s.supplier_id);
  • MySQL: UPDATE ifadesinde dogrudan bir JOIN kullanin. 
    -- MySQL Ornegi
UPDATE products p
JOIN suppliers s ON p.supplier_id = s.supplier_id
SET p.price = s.cost * 1.5
WHERE p.category_id = 10;
  • Standart SQL MERGE: MERGE ifadesi (UPSERT altinda ele alinir) birlesmelere dayali guncellemeleri de isliyebilir.

Guncellenen Degerleri Alma (Sozdizimi Degisir): INSERT'e benzer sekilde, guncellemeden sonra degerleri gormek isteyebilirsiniz. * PostgreSQL/Oracle: RETURNING yan tumcesi. * SQL Server: OUTPUT yan tumcesi. * MySQL: Genellikle sonraki bir SELECT gerektirir.

UPDATE icin En Iyi Uygulamalar: 1. Tum satirlari guncellemeyi kastetmiyorsaniz her zaman WHERE kullanin. 2. Once SELECT ile test edin: UPDATE'i calistirmadan once hangi satirlarin etkilenecegini dogrulamak icin ayni WHERE yan tumcesiyle bir SELECT sorgusu yazin. 3. Islemler (Transaction) Kullanin: Ozellikle karmasik veya birden fazla satiri etkileyen guncellemeleri, guvenlik icin bir islem (BEGIN/COMMIT/ROLLBACK) icine sarin. 4. Performansi Dikkate Alin: Indeksli sutunlar veya buyuk tablolar uzerindeki guncellemeler yavas olabilir. Gerekirse toplu islemlerle guncelleyin.

DELETE Ifadeleri (Veri Silme)

DELETE ifadesi bir tablodan satirlari siler.

Standart SQL Sozdizimi:

DELETE FROM table_name
WHERE condition; -- KRITIK: Hangi satirlarin silinecegini filtreler

Onemli: UPDATE gibi, tum satirlari silmeyi kastetmiyorsaniz her zaman bir WHERE yan tumcesi kullanin. DELETE ifadesinde WHERE'i unutmak son derece tehlikelidir.

Ornekler:

-- Standart SQL: Belirli bir siparisi sil
DELETE FROM orders
WHERE order_id = 9876;

-- Standart SQL: Bir yildir giris yapmayan inaktif kullanicilari sil
DELETE FROM users
WHERE status = "inactive" 
AND last_login_date < CURRENT_DATE - INTERVAL "1" YEAR; -- Aralik sozdizimi degisebilir

Diger Tablolara Dayali Silme (Sozdizimi Degisir): UPDATE'e benzer sekilde, birlesmelere dayali satirlari silmek farkli sozdizimi gerektirir. * PostgreSQL/SQL Server: Bir USING (PostgreSQL) veya FROM (SQL Server) yan tumcesi kullanin. 

-- PostgreSQL Ornegi: 'bad_debt' olarak isaretlenen musterilerden siparisleri sil
DELETE FROM orders o
USING customers c
WHERE o.customer_id = c.customer_id AND c.status = "bad_debt";
* Oracle/Db2/MySQL: Genellikle IN veya EXISTS ile alt sorgular kullanir. 
-- Ornek (Alt Sorgu): 'bad_debt' olarak isaretlenen musterilerden siparisleri sil
DELETE FROM orders
WHERE customer_id IN (SELECT customer_id FROM customers WHERE status = "bad_debt");

Silinen Degerleri Alma (Sozdizimi Degisir): * PostgreSQL/Oracle: RETURNING yan tumcesi. * SQL Server: OUTPUT yan tumcesi. * MySQL: Dogrudan desteklenmez.

TRUNCATE TABLE (Tum Verileri Hizlica Silme)

TRUNCATE TABLE ifadesi bir tablodan tum satirlari cok hizli bir sekilde siler. Genellikle DELETE FROM table_name;'den cok daha hizlidir cunku cogu zaman bireysel satir silme mekanizmalarini atlar.

Standart SQL Sozdizimi:

TRUNCATE TABLE table_name;

Temel Farklar: DELETE vs. TRUNCATE:

Ozellik DELETE FROM table WHERE ... TRUNCATE TABLE table
Hiz Daha Yavas (satirlari tarar/kaydeder) Cok Daha Hizli (genellikle yer ayirir)
WHERE Yan Tumcesi Evet Hayir (Her zaman tum satirlari siler)
Islem Kaydi Her silinen satiri kaydeder Minimal kayit (genellikle)
Tetikleyiciler DELETE tetikleyicilerini calistirir Genellikle tetikleyicileri calistirmaz
Donus Degeri Silinen satirlari dondurabilir Hayir
Izinler DELETE izni gerekli Genellikle daha yuksek yetkiler gerektirir (TRUNCATE, ALTER, DROP)
Kimlik Sifirlama Hayir Genellikle otomatik artis sayaclarini sifirlar (ornegin, PostgreSQL'de RESTART IDENTITY secenegi)
Yabanci Anahtarlar Standart kontroller uygulanir FK'lar tarafindan referans gosteriliyorsa basarisiz olabilir, CASCADE kullanilmadikca (tehlikeli!)

Kullanim Alanlari: TRUNCATE oncelikli olarak yeni veri yuklemeden once gecici veya hazirlama tablolarini hizlica temizlemek icin kullanilir. Kosullu silme, tetikleyicilerin calismasi veya satir duzeyinde kayit gerektiginde DELETE kullanin.

Islem (Transaction) Yonetimi (Standart SQL)

Islemler, bir dizi SQL operasyonunun tek, atomik bir calisma birimi olarak ele alinmasini saglar. Ya islem icindeki tum operasyonlar basarili olur ve kalici hale getirilir (COMMIT), ya da hicbiri (ROLLBACK). Bu, ACID ozelliklerini (Atomiklik, Tutarlilik, Izolasyon, Dayaniklilik) garanti eder.

Standart SQL Komutlari:

  1. BEGIN TRANSACTION (veya genellikle sadece BEGIN): Yeni bir islem baslatir.
  2. COMMIT: BEGIN'den bu yana yapilan tum degisiklikleri kalici hale getirir ve islemi sonlandirir.
  3. ROLLBACK: BEGIN'den bu yana yapilan tum degisiklikleri iptal eder ve islemi sonlandirir.
  4. SAVEPOINT savepoint_name: Bir islem icinde adlandirilmis bir nokta olusturur.
  5. ROLLBACK TO SAVEPOINT savepoint_name: Belirtilen kayit noktasindan sonra yapilan degisiklikleri iptal eder, ancak islemi aktif tutar.

Ornek: Hesaplar arasi para transferi (atomik olarak gerceklesmeli).

BEGIN TRANSACTION;

-- Hesap A'dan tutari cikar
UPDATE accounts
SET balance = balance - 100.00
WHERE account_id = "A123";

-- Hesap B'ye tutari ekle
UPDATE accounts
SET balance = balance + 100.00
WHERE account_id = "B456";

-- Burada hatalari veya kosullari kontrol edin...

-- Her sey yolundaysa, kalici hale getirin
COMMIT;

-- Bir seyler yanlış gittiyse (ornegin, commit'ten once yetersiz bakiye tespit edildiyse),
-- tum degisiklikleri iptal edin:
-- ROLLBACK;

Otomatik Onay (Auto-Commit): Varsayilan olarak, bircok SQL araci ve veritabani baglantisi "otomatik onay" modunda calisir; burada her bireysel SQL ifadesi kendi islemi olarak ele alinir. Coklu ifade islemleri gerceklestirmek icin, genellikle BEGIN ile acikca bir islem baslatmaniz gerekir.

Izolasyon Seviyeleri: Standart SQL, degisikliklerin gorunurlugu konusunda islemlerin birbirleriyle nasil etkilesecegini kontrol eden izolasyon seviyelerini (READ UNCOMMITTED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE) tanimlar. READ COMMITTED en yaygin varsayilandir.

UPSERT Islemleri (INSERT veya UPDATE)

"UPSERT", bir anahtar catismasina dayali olarak bir satir yoksa eklemek veya varsa guncellemek kavramidir. Standart SQL yolu MERGE'dir, ancak belirli veritabanlarinin kendilerine ozgu sozdizimi vardir.

Standart SQL MERGE Ifadesi:

MERGE ifadesi, bir kaynak ve hedef tablo arasindaki eslesen verilere dayali olarak kosullu eklemeler, guncellemeler veya silmeler gerceklestirmek icin standart bir yol saglar. Gucludur ancak karmasik olabilir.

-- Standart MERGE Sozdizimi (Kavramsal)
MERGE INTO target_table AS T
USING source_table AS S
ON (T.match_column = S.match_column)
WHEN MATCHED THEN
    UPDATE SET T.column1 = S.value1, T.column2 = S.value2
WHEN NOT MATCHED THEN
    INSERT (column1, column2, match_column)
    VALUES (S.value1, S.value2, S.match_column);
  • MERGE icin destek ve kesin sozdizimi detaylari veritabanlari arasinda degisir (Oracle, SQL Server, Db2 tarafindan iyi desteklenir; PostgreSQL daha yeni ekledi).

Saticiya Ozgu UPSERT Sozdizimi:

  • PostgreSQL: INSERT ... ON CONFLICT ... DO UPDATE SET ... veya DO NOTHING. 
    -- PostgreSQL Ornegi: Urun envanter sayisini ekle veya guncelle
INSERT INTO product_inventory (product_id, quantity)
VALUES (101, 10)
ON CONFLICT (product_id)
    DO UPDATE SET quantity = product_inventory.quantity + EXCLUDED.quantity;
  • MySQL: INSERT ... ON DUPLICATE KEY UPDATE .... 
    -- MySQL Ornegi: Urun envanter sayisini ekle veya guncelle
INSERT INTO product_inventory (product_id, quantity)
VALUES (101, 10)
ON DUPLICATE KEY UPDATE quantity = quantity + VALUES(quantity);
  • SQLite: INSERT ... ON CONFLICT ... DO UPDATE SET ... (PostgreSQL'e benzer). 
    -- SQLite Ornegi: Urun envanter sayisini ekle veya guncelle
INSERT INTO product_inventory (product_id, quantity)
VALUES (101, 10)
ON CONFLICT (product_id)
    DO UPDATE SET quantity = product_inventory.quantity + EXCLUDED.quantity;

Kullanim Alanlari: UPSERT/MERGE, veri senkronizasyonu, harici kaynaklardan veri yukleme, sayaclari guncelleme veya yapilandirma tablolarini korumak icin cok kullanislidir.

Bu DML (Veri Manipulasyon Dili) ifadelerini ve islem kontrolunu anlamak, veritabanlariyla guvenilir bir sekilde etkilesen uygulamalar olusturmak icin temeldir.
