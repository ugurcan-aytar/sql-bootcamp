Gun 1: Uygulamali Calistay

Odoo Veritabanini Kesfetme

Semalarda ve Tablolarda Gezinme

Bu uygulamali calistayda, gun boyunca ogrendigimiz SQL kavramlarini uygulayarak Odoo veritabaninin yapisini ve icerigini kesfedecegiz. Odoo veritabani, veri modelini etkili bir sekilde duzenlemek icin PostgreSQL'in sema yeteneklerini kullanir.

Odoo'nun Sema Organizasyonunu Anlama:

Odoo genellikle asagidaki sema yapisini kullanir: - public: Temel uygulama tablolarini icerir - Belirli moduller veya uzantilar icin ek semalar olusturulabilir

Mevcut semalari ve tablolari kesfederek baslayalim:

-- Veritabanindaki tum semalari listele
SELECT schema_name 
FROM information_schema.schemata
ORDER BY schema_name;
-- Public semasindaki tablolari listele
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
-- Her semadaki tablo sayisini say
SELECT table_schema, COUNT(*) as table_count
FROM information_schema.tables
WHERE table_type = 'BASE TABLE'
GROUP BY table_schema
ORDER BY table_count DESC;

Tablo Yapisini Inceleme:

Odoo veritabanindaki bazi onemli tablolarin yapisini inceleyelim:

-- res_partner tablosunu tanimla (kisiler/musteriler)
SELECT 
    column_name, 
    data_type, 
    character_maximum_length,
    is_nullable
FROM 
    information_schema.columns
WHERE 
    table_name = 'res_partner'
ORDER BY 
    ordinal_position;

-- sale_order tablosundaki kisitlamalari incele
SELECT 
    tc.constraint_name, 
    tc.constraint_type, 
    kcu.column_name,
    ccu.table_name AS foreign_table,
    ccu.column_name AS foreign_column
FROM 
    information_schema.table_constraints tc
JOIN 
    information_schema.key_column_usage kcu
    ON tc.constraint_name = kcu.constraint_name
LEFT JOIN 
    information_schema.constraint_column_usage ccu
    ON ccu.constraint_name = tc.constraint_name
WHERE 
    tc.table_name = 'sale_order'
ORDER BY 
    tc.constraint_type, 
    kcu.column_name;

Tablolar Arasindaki Iliskileri Bulma:

Tablo iliskilerini anlamak etkili sorgulama icin cok onemlidir:

-- res_partner'a referans veren tum tablolari bul
SELECT 
    tc.table_name, 
    kcu.column_name
FROM 
    information_schema.table_constraints tc
JOIN 
    information_schema.key_column_usage kcu
    ON tc.constraint_name = kcu.constraint_name
JOIN 
    information_schema.constraint_column_usage ccu
    ON ccu.constraint_name = tc.constraint_name
WHERE 
    tc.constraint_type = 'FOREIGN KEY' AND
    ccu.table_name = 'res_partner' AND
    ccu.column_name = 'id'
ORDER BY 
    tc.table_name;

ERP Veri Modelini Anlama

Odoo'nun veri modeli kapsamli ve birbirine baglidir. Bazi temel is varliklarini ve iliskilerini inceleyelim:

Temel Is Varliklari:

  1. Partnerler (res_partner):
    • Musterileri, tedarikcileri, calisanlari ve diger kisileri temsil eder
    • Bircok is surecinin merkezinde yer alir
  2. Urunler (product_template, product_product):
    • product_template: Urun tanimi (ad, kategori, vb.)
    • product_product: Urun varyantlari (belirli yapilandirmalar)
  3. Satislar (sale_order, sale_order_line):
    • sale_order: Satis siparisleri icin baslik bilgileri
    • sale_order_line: Satis siparisleri icindeki kalem satirlari
  1. Envanter (stock_warehouse, stock_location, stock_quant):
    • stock_warehouse: Fiziksel veya mantiksal depolar
    • stock_location: Depolar icindeki konumlar
    • stock_quant: Gercek envanter miktarlari
  2. Muhasebe (account_move, account_move_line):
    • account_move: Yevmiye kayitlari, faturalar
    • account_move_line: Yevmiye kayitlarindaki bireysel kalem satirlari

Bu varliklari bazi sorgularla inceleyelim:

-- Partnerlerin ture gore genel gorunumu
SELECT 
    CASE 
        WHEN customer_rank > 0 THEN 'Customer'
        WHEN supplier_rank > 0 THEN 'Supplier'
        WHEN employee = true THEN 'Employee'
        ELSE 'Other'
    END AS partner_type,
    COUNT(*) as count
FROM 
    res_partner
GROUP BY 
    partner_type
ORDER BY 
    count DESC;
-- Urun kategorileri hiyerarsisi
WITH RECURSIVE category_tree AS (
    SELECT 
        id, 
        name, 
        parent_id, 
        0 AS level,
        name AS path
    FROM product_category
    WHERE parent_id IS NULL
    UNION ALL
    SELECT 
        pc.id, 
        pc.name, 
        pc.parent_id, 
        ct.level + 1,
        ct.path || ' > ' || pc.name
    FROM product_category pc 
    JOIN category_tree ct ON pc.parent_id = ct.id 
)
SELECT 
    level,
    path,
    (SELECT COUNT(*) FROM product_template pt WHERE pt.categ_id = category_tree.id) AS product_count
FROM category_tree
ORDER BY path;

Odoo Veritabanina Temel Sorgular Yazma

Simdi faydali is bilgileri cikaran sorgular yazmayi pratik edelim:

Alistirma 1: Musteri Analizi

-- Satis hacmine gore en iyi 10 musteri
SELECT 
    rp.name AS customer,
    rp.email,
    rp.phone,
    COUNT(so.id) AS order_count,
    SUM(so.amount_total) AS total_sales
FROM 
    res_partner rp
JOIN 
    sale_order so ON rp.id = so.partner_id
WHERE 
    so.state IN ('sale', 'done')
GROUP BY 
    rp.id, rp.name, rp.email, rp.phone
ORDER BY 
    total_sales DESC
LIMIT 10;
-- Son 6 ayda siparis vermeyen musteriler
SELECT 
    rp.name,
    rp.email,
    rp.phone,
    MAX(so.date_order) AS last_order_date
FROM 
    res_partner rp
LEFT JOIN 
    sale_order so ON rp.id = so.partner_id AND so.state IN ('sale', 'done')
WHERE 
    rp.customer_rank > 0
GROUP BY 
    rp.id, rp.name, rp.email, rp.phone
HAVING 
    MAX(so.date_order) < CURRENT_DATE - INTERVAL '6 months'
    OR MAX(so.date_order) IS NULL
ORDER BY 
    last_order_date DESC NULLS FIRST;

Alistirma 2: Urun Performansi

-- En cok satan urunler
SELECT 
    pt.name AS product,
    pc.name AS category,
    SUM(sol.product_uom_qty) AS quantity_sold,
    SUM(sol.price_subtotal) AS revenue
FROM 
    sale_order_line sol
JOIN 
    sale_order so ON sol.order_id = so.id
JOIN 
    product_product pp ON sol.product_id = pp.id
JOIN 
    product_template pt ON pp.product_tmpl_id = pt.id
JOIN 
    product_category pc ON pt.categ_id = pc.id
WHERE 
    so.state IN ('sale', 'done')
GROUP BY 
    pt.id, pt.name, pc.name
ORDER BY 
    revenue DESC
LIMIT 15;

-- Dusuk stoklu urunler
SELECT 
    pt.name AS product,
    pt.list_price,
    COALESCE(SUM(sq.quantity), 0) AS current_stock,
    swo.product_min_qty AS minimum_stock,
    CASE 
        WHEN COALESCE(SUM(sq.quantity), 0) < swo.product_min_qty THEN 'Reorder'
        ELSE 'OK'
    END AS stock_status
FROM 
    product_template pt
JOIN product_product pp ON pp.product_tmpl_id = pt.id
LEFT JOIN stock_quant sq ON pp.id = sq.product_id
LEFT JOIN stock_warehouse_orderpoint swo ON pp.id = swo.product_id
WHERE pt.active = true AND pt.type = 'consu'
GROUP BY 
    pt.id, pt.name, pt.list_price, swo.product_min_qty
HAVING 
    COALESCE(SUM(sq.quantity), 0) < swo.product_min_qty
ORDER BY 
    current_stock ASC;

Alistirma 3: Satis Analizi

-- Aylik satis trendi
SELECT 
    DATE_TRUNC('month', so.date_order) AS month,
    COUNT(DISTINCT so.id) AS order_count,
    SUM(so.amount_total) AS total_sales
FROM 
    sale_order so
WHERE 
    so.state IN ('sale', 'done')
    AND so.date_order >= CURRENT_DATE - INTERVAL '12 months'
GROUP BY 
    DATE_TRUNC('month', so.date_order)
ORDER BY 
    month;

-- Ulkeye gore satislar
SELECT 
    c.name AS country,
    COUNT(DISTINCT so.id) AS order_count,
    SUM(so.amount_total) AS total_sales
FROM 
    sale_order so
JOIN 
    res_partner rp ON so.partner_id = rp.id
JOIN 
    res_country c ON rp.country_id = c.id
WHERE 
    so.state IN ('sale', 'done')
GROUP BY 
    c.name
ORDER BY 
    total_sales DESC;

Alistirma 4: Envanter Degerlendirmesi

-- Depoya gore envanter degeri
SELECT 
    sw.name AS warehouse,
    SUM(sq.quantity * CAST(pp.standard_price->'1' AS NUMERIC )) AS inventory_value,
    COUNT(DISTINCT pp.id) AS product_count
FROM 
    stock_quant sq
JOIN 
    stock_location sl ON sq.location_id = sl.id
JOIN 
    stock_warehouse sw ON sl.company_id = sw.company_id
JOIN 
    product_product pp ON sq.product_id = pp.id
JOIN 
    product_template pt ON pp.product_tmpl_id = pt.id
WHERE 
    sl.usage = 'internal'
GROUP BY 
    sw.name
ORDER BY 
    inventory_value DESC;

Gercek Verilerle Pratik Alistirmalar

Simdi rehberli alistirmalarla pratik yapma zamani. Her alistirma icin, cozume bakmadan once sorguyu kendiniz yazmayi deneyin.

Alistirma 1: Odenmemis Faturalari Bulma

30 gundan fazla gecmis ve odenmemis tum musteri faturalarini bulan bir sorgu yazin. Musteri adi, fatura numarasi, fatura tarihi, vade tarihi ve odenmesi gereken tutari dahil edin.

Ipuclari

  • Tum odenmemis faturalari bulmak icin account_move tablosunu kullanin.
  • Musteri adini bulmak icin res_partner tablosunu kullanin.

Alistirma 1: Odenmemis Faturalari Bulma - Cozum 

SELECT 
    rp.name AS customer,
    am.name AS invoice_number,
    am.invoice_date,
    am.invoice_date_due AS due_date,
    CURRENT_DATE - am.invoice_date_due AS days_overdue,
    am.amount_residual AS amount_due
FROM 
    account_move am
JOIN 
    res_partner rp ON am.partner_id = rp.id
WHERE 
    am.move_type = 'out_invoice'
    AND am.state = 'posted'
    AND am.payment_state != 'paid'
    AND am.invoice_date_due < CURRENT_DATE - INTERVAL '30 days'
ORDER BY 
    days_overdue DESC;

Alistirma 2: Urun Karlilik Analizi

Son yil icinde satilan her urun icin kar marjini hesaplayan bir sorgu yazin. Urun adi, kategori, toplam satilan miktar, toplam gelir, toplam maliyet ve kar marji yuzdesini dahil edin.

Ipuclari

  • sale_order_line tablosunu kullanin
  • sale_order tablosunu kullanin
  • product_product tablosunu kullanin
  • product_template tablosunu kullanin
  • product_category tablosunu kullanin

Alistirma 2: Urun Karlilik Analizi - Cozum 

SELECT 
    pt.name AS product,
    pc.name AS category,
    SUM(sol.product_uom_qty) AS quantity_sold,
    SUM(sol.price_subtotal) AS revenue,
    SUM(sol.product_uom_qty * CAST(pp.standard_price->'1' AS NUMERIC)) AS cost,
    SUM(sol.price_subtotal) - SUM(sol.product_uom_qty * CAST(pp.standard_price->'1' AS NUMERIC)) AS profit,
    CASE 
        WHEN SUM(sol.price_subtotal) = 0 THEN 0
        ELSE ROUND((SUM(sol.price_subtotal) - SUM(sol.product_uom_qty * CAST(pp.standard_price->'1' AS NUMERIC))) / SUM(sol.price_subtotal) * 100, 2)
    END AS profit_margin_percentage
FROM 
    sale_order_line sol
JOIN 
    sale_order so ON sol.order_id = so.id
JOIN 
    product_product pp ON sol.product_id = pp.id
JOIN 
    product_template pt ON pp.product_tmpl_id = pt.id
JOIN 
    product_category pc ON pt.categ_id = pc.id
WHERE 
    so.state IN ('sale', 'done')
    AND so.date_order >= CURRENT_DATE - INTERVAL '1 year'
GROUP BY 
    pt.id, pt.name, pc.name
ORDER BY 
    profit_margin_percentage DESC;

Alistirma 3: Musteri Segmentasyonu

Musterileri satin alma davranislarina gore segmentlere ayiran bir sorgu yazin. Su segmentleri olusturun: - VIP musteriler (10.000 $'dan fazla satin alma) - Duzenli musteriler (1.000 $ ile 10.000 $ arasi satin alma) - Ara sira alisveri yapan musteriler (1.000 $'dan az satin alma) - Aktif olmayan musteriler (satin alma yok)

Alistirma 3: Musteri Segmentasyonu - Cozum 

SELECT 
customer_segment,
COUNT(*) AS customer_count
FROM (
    SELECT 
    CASE 
    WHEN total_spent > 10000 THEN 'VIP Customer'
    WHEN total_spent BETWEEN 1000 AND 10000 THEN 'Regular Customer'
    WHEN total_spent < 1000 AND total_spent > 0 THEN 'Occasional Customer'
    ELSE 'Inactive Customer'
    END AS customer_segment
    FROM(
        SELECT
        rp.id,
        COALESCE(SUM(so.amount_total), 0) AS total_spent
        FROM
        res_partner rp
        LEFT JOIN
        sale_order so
        ON rp.id=so.partner_id
        AND so.state IN('sale', 'done')
        GROUP BY
        rp.id
    ) AS spending_data
) AS segmented_data
GROUP BY
customer_segment
ORDER BY
MIN(CASE customer_segment
    WHEN 'VIP Customer' THEN 1
    WHEN 'Regular Customer' THEN 2
    WHEN 'Occasional Customer' THEN 3
    ELSE 4
    END)

Alistirma 4: Envanter Devir Hizi

Son yil icinde her urun kategorisi icin envanter devir hizini hesaplayan bir sorgu yazin. Envanter devir hizi (Satilan Mallarin Maliyeti / Ortalama Envanter) olarak hesaplanir.

Alistirma 4: Envanter Devir Hizi - Cozum 

WITH sold_items AS (
    SELECT 
        pt.categ_id,
        SUM(sol.product_uom_qty * pt.standard_price) AS cost_of_goods_sold
    FROM 
        sale_order_line sol
    JOIN 
        sale_order so ON sol.order_id = so.id
    JOIN 
        product_product pp ON sol.product_id = pp.id
    JOIN 
        product_template pt ON pp.product_tmpl_id = pt.id
    WHERE 
        so.state IN ('sale', 'done')
        AND so.date_order >= CURRENT_DATE - INTERVAL '1 year'
    GROUP BY 
        pt.categ_id
),
current_inventory AS (
    SELECT 
        pt.categ_id,
        SUM(sq.quantity * pt.standard_price) AS inventory_value
    FROM 
        stock_quant sq
    JOIN 
        product_product pp ON sq.product_id = pp.id
    JOIN 
        product_template pt ON pp.product_tmpl_id = pt.id
    JOIN 
        stock_location sl ON sq.location_id = sl.id
    WHERE 
        sl.usage = 'internal'
    GROUP BY 
        pt.categ_id
)
SELECT 
    pc.name AS category,
    COALESCE(si.cost_of_goods_sold, 0) AS cost_of_goods_sold,
    COALESCE(ci.inventory_value, 0) AS current_inventory_value,
    CASE 
        WHEN COALESCE(ci.inventory_value, 0) = 0 THEN NULL
        ELSE ROUND(COALESCE(si.cost_of_goods_sold, 0) / COALESCE(ci.inventory_value, 0), 2)
    END AS inventory_turnover
FROM 
    product_category pc
LEFT JOIN 
    sold_items si ON pc.id = si.categ_id
LEFT JOIN 
    current_inventory ci ON pc.id = ci.categ_id
WHERE 
    COALESCE(si.cost_of_goods_sold, 0) > 0 OR COALESCE(ci.inventory_value, 0) > 0
ORDER BY 
    inventory_turnover DESC NULLS LAST;

Alistirma 5: JSON Verileriyle Calisma

Odoo bazi verileri JSON formatinda saklar. Bir JSON sutunundan veri cikarmak ve analiz etmek icin bir sorgu yazin. Ornegin, bir JSON sutununda saklanan belirli ozelliklere sahip tum urunleri bulun.

Cozum 

-- Urun ozelliklerinin 'attributes_json' adli bir JSONB sutununda saklandigi varsayilarak
SELECT 
    pt.name AS product,
    pt.attributes_json->'color' AS color,
    pt.attributes_json->'size' AS size
FROM 
    product_template pt
WHERE 
    pt.attributes_json ? 'color'
    AND pt.attributes_json->>'color' = 'red'
ORDER BY 
    pt.name;

-- Ozellikler farkli bir yapida ise alternatif yaklasim
SELECT 
    pt.name AS product,
    jsonb_extract_path_text(pt.attributes_json, 'specifications', 'dimensions', 'height') AS height,
    jsonb_extract_path_text(pt.attributes_json, 'specifications', 'dimensions', 'width') AS width
FROM 
    product_template pt
WHERE 
    pt.attributes_json IS NOT NULL
    AND jsonb_extract_path_text(pt.attributes_json, 'specifications', 'material') = 'aluminum'
ORDER BY 
    pt.name;

Sonuc ve Sonraki Adimlar

Bu uygulamali calistayda, Odoo veritabani yapisini kesfettiniz ve degerli is bilgileri cikarmak icin SQL sorgulari yazmayi pratik ettiniz. Bu beceriler, urun sahibi olarak rolunuz icin onemli olacak ve size su konularda yardimci olacaktir:

  1. ERP sisteminizin altinda yatan veri modelini anlama
  2. Karar verme icin gereken belirli bilgileri cikartma
  3. Gercek verileri inceleyerek sistem davranisini dogrulama
  4. Belirli is sorularini yanitleyan ozel raporlar olusturma
  5. Teknik ekip uyeleriyle daha etkili iletisim kurma

Yarin, PostgreSQL'in guclu JSON yeteneklerine ozel bir odaklanma ile ileri sorgulama teknikleri ve veri manipulasyonunu kesfederek bu temeller uzerine insa edecegiz.
