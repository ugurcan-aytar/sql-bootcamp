Gun 2: JSON ile Calisma

Oturum 4: JSON Verileri ile Calisma (2 saat)

JSON (JavaScript Object Notation), veri degisimi icin standart bir format haline gelmistir ve iliskisel veritabanlarinda yari yapilandirilmis verilerin saklanmasi icin giderek daha fazla kullanilmaktadir. Modern RDBMS'lerin cogu JSON verilerini saklama, sorgulama ve indeksleme yollari sunmaktadir, ancak belirli ozellikler, veri turleri ve sozdizimi farklilik gostermektedir.

JSON Verilerini Saklama: Yaygin Yaklasimlar

Veritabanlari JSON depolamayi farkli sekillerde ele alir:

  1. Ozel JSON Veri Turleri: Optimize edilmis depolama ve sorgulama sunar.
    • PostgreSQL: JSON (tam metni saklar, bosluk/siralama/tekrarlari korur) ve JSONB (ikili format, sorgu/indeksleme icin optimize edilmis, bosluk/tekrarlari kaldirir, sirayi korumaz - genellikle tercih edilir).
    • MySQL: JSON turu (5.7'den beri optimize edilmis ikili format olarak uygulanmistir).
    • Oracle: JSON turu (dahili olarak VARCHAR2, CLOB veya BLOB olarak saklanabilir, belirli kisitlamalarla).
    • SQLite: JSON TEXT olarak saklanir, ancak JSON fonksiyonlari mevcuttur.
  2. JSON Fonksiyonlari ile Metin Tabanli Depolama: JSON'u TEXT, VARCHAR(MAX), CLOB vb. olarak saklayin ve ayristirmak ve sorgulamak icin yerlesik fonksiyonlari kullanin.
    • SQL Server: NVARCHAR(MAX) kullanir ve kapsamli fonksiyonlar saglar (JSON_VALUE, JSON_QUERY, OPENJSON, ISJSON, vb.).
    • Db2: Genellikle VARCHAR, CLOB veya BLOB icinde saklanir, etkilesim icin JSON SQL fonksiyonlari ile birlikte.

Yaklasim Secimi: Mevcut oldugunda ozel JSON turlerini (PostgreSQL JSONB veya MySQL JSON gibi) kullanmak genellikle onerilir, cunku performans avantajlari ve ozellestirilmis indeksleme sunarlar.

JSON Verilerini Sorgulama: Operatorler ve Fonksiyonlar

JSON sutunlarindan veri cikarma sozdizimi onemli olcude degisir.

1. Yol Navigasyonu: Bir JSON nesnesi veya dizisi icindeki belirli degerlere erisim.

  • PostgreSQL: -> (ogesi/alani JSON/JSONB olarak al), ->> (metin olarak al), #> (yoldaki oge/alani JSON/JSONB olarak al), #>> (yolda metin olarak al) operatorlerini kullanir. 
    -- PostgreSQL Ornekleri
SELECT data_column->'user'->>'name' FROM logs WHERE id = 1;
SELECT data_column#>>'{address,city}' FROM users WHERE id = 1;
SELECT data_column->'tags'->>0 FROM posts WHERE id = 1; -- Ilk dizi ogesine erisim
  • MySQL: -> (JSON_EXTRACT ile esdeger), ->> (cikarir ve tirnaklari kaldirir) kullanir. 
    -- MySQL Ornekleri
SELECT data_column->'$.user.name' FROM logs WHERE id = 1;
SELECT data_column->>'$.user.name' FROM users WHERE id = 1;
SELECT data_column->>'$.tags[0]' FROM posts WHERE id = 1; -- Ilk dizi ogesine erisim
  • SQL Server: JSON_VALUE (skaler degerler icin) ve JSON_QUERY (nesneler/diziler icin) kullanir. 
    -- SQL Server Ornekleri
SELECT JSON_VALUE(data_column, '$.user.name') FROM logs WHERE id = 1;
SELECT JSON_QUERY(data_column, '$.address') FROM users WHERE id = 1;
SELECT JSON_VALUE(data_column, '$.tags[0]') FROM posts WHERE id = 1;
  • Oracle: JSON sutunlariyla nokta notasyonu veya JSON_VALUE, JSON_QUERY kullanir. 
    -- Oracle Ornekleri ('data_column' JSON turunde oldugu varsayilarak)
SELECT l.data_column.user.name FROM logs l WHERE id = 1;
SELECT JSON_VALUE(data_column, '$.address.city') FROM users WHERE id = 1;
SELECT JSON_VALUE(data_column, '$.tags[0]') FROM posts WHERE id = 1;
  • Standart SQL/JSON Yolu: SQL/JSON yol dili ($ kok icin, . uyeler icin, [] diziler icin) ile JSON_VALUE ve JSON_QUERY fonksiyonlari daha standart hale gelmektedir.

2. Varlik ve Tur Kontrolleri:

  • PostgreSQL (JSONB): ? (anahtar var mi), ?| (herhangi bir anahtar var mi), ?& (tum anahtarlar var mi). 
    SELECT * FROM users WHERE data_column ? 'address';-- PostgreSQL Ornegi
  • MySQL: JSON_CONTAINS_PATH, JSON_KEYS, JSON_TYPE. 
    SELECT * FROM users WHERE JSON_CONTAINS_PATH(data_column, 'one', '$.address');-- MySQL Ornegi
  • SQL Server: ISJSON fonksiyonu, yol yoksa JSON_VALUE NULL dondurur. 
    SELECT * FROM users WHERE ISJSON(data_column) > 0 AND JSON_VALUE(data_column, '$.address') IS NOT NULL;-- SQL Server Ornegi
  • Oracle: IS JSON kosulu, JSON_EXISTS fonksiyonu. 
    SELECT * FROM users WHERE data_column IS JSON AND JSON_EXISTS(data_column, '$.address');-- Oracle Ornegi

3. JSON Icinde Arama (Kapsama):

  • PostgreSQL (JSONB): @> (kapsar), <@ (kapsanir). 
    SELECT * FROM users WHERE data_column->'tags' @> '"admin"'::jsonb;
-- PostgreSQL Ornegi: Belirli bir etikete sahip kullanicilari bul
SELECT * FROM users WHERE data_column <@ '{"roles": ["editor"], "status": "active"}'::jsonb;
  • MySQL: JSON_CONTAINS, JSON_SEARCH. 
    SELECT * FROM users WHERE JSON_CONTAINS(data_column->'$.tags', '"admin"');-- MySQL Ornegi: Belirli bir etikete sahip kullanicilari bul
  • SQL Server/Oracle: Genellikle degerleri cikarmak veya yol varlik kontrollerini kullanmak gerektirir.

4. Diziler ve Nesneler icin Fonksiyonlar: Dizi uzunlugunu, nesne anahtarlarini almak, dizileri/nesneleri satirlara genisletmek icin fonksiyonlar yaygindir ancak degiskenlik gosterir.

  • PostgreSQL: jsonb_array_length, jsonb_object_keys, jsonb_array_elements, jsonb_each.
  • MySQL: JSON_LENGTH, JSON_KEYS, JSON_TABLE (JSON'u iliskisel satirlara donusturmek icin guclu fonksiyon).
  • SQL Server: JSON_QUERY (diziler icin), OPENJSON (JSON'u iliskisel satirlara donusturmek icin cok guclu fonksiyon).
  • Oracle: JSON_TABLE (standart ve guclu).

JSON Verilerini Indeksleme

Buyuk miktarda JSON verisini verimli bir sekilde sorgulamak indeksleme gerektirir. Stratejiler, veritabani sistemine ve depolama formatina buyuk olcude baglidir.

  1. Belirli Alanlari Indeksleme: Fonksiyonlar veya yol operatorleri kullanarak JSON'dan cikartilan skaler degerler uzerinde standart B-tree indeksleri olusturun. Bu, bilinen, sik sorgulanan alanlar uzerinde esitlik veya aralik sorgulari icin iyi calisir.

    -- PostgreSQL Ornegi
CREATE INDEX idx_user_email ON users ((data_column->>'email'));

-- MySQL Ornegi (Uretilmis Sutun)
ALTER TABLE users ADD COLUMN email_virtual VARCHAR(100) 
    GENERATED ALWAYS AS (data_column->>'$.email') STORED;
CREATE INDEX idx_user_email ON users (email_virtual);

-- SQL Server Ornegi (Hesaplanmis Sutun)
ALTER TABLE users ADD email_computed AS JSON_VALUE(data_column, '$.email');
CREATE INDEX idx_user_email ON users (email_computed);

  2. Tam JSON Indeksleme (GIN/Ters Indeksler): Kapsama (@>) ve varlik (?) kontrolleri dahil cesitli sorgulari desteklemek icin tum JSON yapisini indeksleyin. Genellikle ikili JSON turleri ile kullanilir.

    • PostgreSQL: USING GIN (jsonb_column) veya USING GIN (jsonb_column jsonb_path_ops).
    • MySQL: JSON dizilerinde cok degerli indeksler veya fonksiyonel indeksler.
    • Oracle: JSON_SEARCH indeksi, fonksiyonel indeksler.
    • SQL Server: Tam metin indeksleri bazen kullanilabilir veya yollari temsil eden hesaplanmis sutunlar uzerinde indeksler.

Dogru indeksleme stratejisini secmek, optimize etmeniz gereken belirli sorgulara baglidir.

JSON Donusturme ve Degistirme

Veritabanlari JSON verileri olusturmak, degistirmek ve donusturmek icin fonksiyonlar saglar.

1. JSON Olusturma: Iliskisel verilerden JSON nesneleri veya dizileri olusturma. * PostgreSQL: jsonb_build_object, jsonb_build_array, jsonb_agg, jsonb_object_agg. * MySQL: JSON_OBJECT, JSON_ARRAY, JSON_ARRAYAGG, JSON_OBJECTAGG. * SQL Server: SELECT ifadelerinde FOR JSON PATH veya FOR JSON AUTO cumleleri. * Oracle: JSON_OBJECT, JSON_ARRAY, JSON_OBJECTAGG, JSON_ARRAYAGG.

2. JSON Degistirme: Bir JSON belgesinin bazi kisimlarini guncelleme. * PostgreSQL: jsonb_set fonksiyonu, || operatoru (birlestir), - operatoru (anahtar sil), #- operatoru (yol sil). * MySQL: JSON_SET, JSON_INSERT, JSON_REPLACE, JSON_REMOVE. * SQL Server: JSON_MODIFY. * Oracle: JSON_MERGEPATCH, JSON_TRANSFORM.

3. JSON'u Iliskisele Donusturme: JSON yapilarini satirlara ve sutunlara genisletme. * PostgreSQL: jsonb_array_elements, jsonb_each, jsonb_to_record, jsonb_to_recordset. * MySQL: JSON_TABLE. * SQL Server: OPENJSON. * Oracle: JSON_TABLE.

Kullanim Alanlari ve En Iyi Uygulamalar

RDBMS'de JSON icin Yaygin Kullanim Alanlari: - Esnek veya gelisen ozellikleri saklama (orn. urun spesifikasyonlari, kullanici tercihleri). - Degisken yapilara sahip olaylari kaydetme. - Yapilandirma verilerini saklama. - Dis API'lerden gelen verileri onbellege alma. - Web on uclerinden gelen verileri isleme.

En Iyi Uygulamalar: - Mumkun Oldugunda Ikili Formatlari Kullanin (JSONB, MySQL JSON): Sorgulama icin genellikle daha iyi performans ve indeksleme. - Uygun Oldugunda Normalize Edin: JSON'u asiri kullanmayin. Bir veri parcasi sabit bir yapiya sahipse, sik sorgulaniyor veya bagimsiz olarak guncelleniyorsa veya guclu iliskisel butunluk (yabanci anahtarlar) gerektiriyorsa, muhtemelen kendi sutununa veya tablosuna aittir. - Stratejik Olarak Indeksleyin: Sorgu kaliplarina gore belirli, sik sorgulanan yollari indeksleyin veya tam JSON indeksleri (GIN gibi) kullanin. - JSON'u Dogrulayin: Saklanan verilerin gecerli JSON oldugunu saglamak icin veritabani kisitlamalarini (IS JSON kontrolleri) veya uygulama mantigi kullanin. - Sozdizimi Farkliliklarini Anlayin: JSON fonksiyonlari ve operatorlerinin yuksek oranda satici ozelinde oldugunu bilin.

JSON verileriyle etkili bir sekilde calismak, NoSQL'in esnekligini iliskisel veritabanlarinin saglamligi ve sorgulama gucuyle birlestirir, ancak sectiginiz RDBMS'nin belirli yeteneklerini ve sozdizimini anlamayi gerektirir.
